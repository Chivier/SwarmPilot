# Quick Start

## Prerequisites

- Python >= 3.11
- [uv](https://docs.astral.sh/uv/) package manager
- (Optional) A running Planner service for cluster integration
- (Optional) A running Predictor with trained models for prediction-based scheduling

## Installation

```bash
cd scheduler
uv sync
```

## Configuration

All settings are controlled via environment variables. Default values work for standalone mode with adaptive bootstrap scheduling.

### Server

| Variable | Default | Description |
|---|---|---|
| `SCHEDULER_HOST` | `0.0.0.0` | Bind address |
| `SCHEDULER_PORT` | `8000` | Bind port |
| `SCHEDULER_ENABLE_CORS` | `true` | Enable CORS middleware |

### Scheduling

| Variable | Default | Description |
|---|---|---|
| `SCHEDULING_STRATEGY` | `adaptive_bootstrap` | Scheduling algorithm: `adaptive_bootstrap`, `probabilistic`, `min_time`, `round_robin`, `random`, `po2`, `serverless` |
| `SCHEDULING_PROBABILISTIC_QUANTILE` | `0.9` | Target quantile for probabilistic scheduling (0.0 - 1.0) |

### Predictor

| Variable | Default | Description |
|---|---|---|
| `PREDICTOR_STORAGE_DIR` | `models` | Directory for trained predictor model files |
| `PREDICTOR_CACHE_MAX_SIZE` | `100` | Maximum number of models to cache in memory |

### Preprocessor

| Variable | Default | Description |
|---|---|---|
| `PREPROCESSOR_CONFIG_FILE` | *(empty)* | Path to preprocessor rules JSON (empty = no preprocessors) |
| `PREPROCESSOR_STRICT` | `true` | Fail if configured preprocessor model is unavailable |

### Training

| Variable | Default | Description |
|---|---|---|
| `TRAINING_ENABLE_AUTO` | `false` | Enable automatic online training |
| `TRAINING_BATCH_SIZE` | `100` | Batch size for training data collection |
| `TRAINING_FREQUENCY` | `3600` | Training frequency in seconds |
| `TRAINING_MIN_SAMPLES` | `10` | Minimum samples required before training |
| `TRAINING_PREDICTION_TYPES` | `quantile` | Prediction types to train (comma-separated) |

### Planner Reporting

| Variable | Default | Description |
|---|---|---|
| `PLANNER_URL` | *(empty)* | Planner service URL for throughput reporting (empty = disabled) |
| `SCHEDULER_AUTO_REPORT` | `0` | Auto-report interval in seconds (0 = disabled) |
| `PLANNER_REPORT_TIMEOUT` | `5.0` | HTTP timeout for report requests (seconds) |

### Planner Registration

| Variable | Default | Description |
|---|---|---|
| `PLANNER_REGISTRATION_URL` | *(empty)* | Planner URL for startup registration (empty = disabled) |
| `SCHEDULER_MODEL_ID` | *(empty)* | Model ID this scheduler handles |
| `SCHEDULER_SELF_URL` | *(empty)* | Advertised URL for this scheduler (how planner reaches us) |
| `PLANNER_REGISTRATION_TIMEOUT` | `10.0` | HTTP timeout for registration calls (seconds) |
| `PLANNER_REGISTRATION_MAX_RETRIES` | `3` | Maximum retries for registration |
| `PLANNER_REGISTRATION_RETRY_DELAY` | `5.0` | Delay between retries (seconds) |

### Proxy

| Variable | Default | Description |
|---|---|---|
| `PROXY_ENABLED` | `true` | Enable the transparent proxy router |
| `PROXY_TIMEOUT` | `300.0` | Timeout for proxy requests (seconds) |
| `WORKER_HTTP_TIMEOUT` | `300.0` | HTTP timeout for worker queue threads (seconds) |

### Logging

| Variable | Default | Description |
|---|---|---|
| `SCHEDULER_LOGURU_LEVEL` | `INFO` | Log level: TRACE, DEBUG, INFO, SUCCESS, WARNING, ERROR, CRITICAL |
| `SCHEDULER_LOG_DIR` | `logs` | Directory for log files |
| `SCHEDULER_ENABLE_JSON_LOGS` | `false` | Enable JSON structured logging |

## Starting the Service

### Standalone

```bash
cd scheduler
uv run uvicorn src.api:app --host 0.0.0.0 --port 8000
```

### With Planner Integration

```bash
PLANNER_REGISTRATION_URL=http://planner:8000/v1/scheduler/register \
SCHEDULER_MODEL_ID=my-model \
SCHEDULER_SELF_URL=http://scheduler:8000 \
PLANNER_URL=http://planner:8000 \
SCHEDULER_AUTO_REPORT=30 \
uv run uvicorn src.api:app --host 0.0.0.0 --port 8000
```

## First Task Submission

### 1. Register an instance

```bash
curl -X POST http://localhost:8000/v1/instance/register \
  -H "Content-Type: application/json" \
  -d '{
    "instance_id": "worker-1",
    "model_id": "llama-7b",
    "endpoint": "http://worker-1:8001",
    "platform_info": {
      "software_name": "vllm",
      "software_version": "0.4.0",
      "hardware_name": "A100"
    }
  }'
```

### 2. Submit a task

```bash
curl -X POST http://localhost:8000/v1/task/submit \
  -H "Content-Type: application/json" \
  -d '{
    "task_id": "task-001",
    "model_id": "llama-7b",
    "task_input": {"prompt": "Hello, world!"},
    "metadata": {"max_tokens": 128}
  }'
```

### 3. Check the result

```bash
curl "http://localhost:8000/v1/task/info?task_id=task-001"
```
