# Algorithm Details

## Problem Formulation

Given:
- A task with `model_id` and `metadata` (describing input characteristics)
- A set of **K** available instances, each with `instance_id`, `endpoint`, and `platform_info`
- A predictor that estimates execution time per (model, platform, metadata) triple
- Per-instance queue state tracking remaining work in the instance's pipeline

Select the instance that minimizes the total expected completion time (queue wait + task execution).

## Queue Information Models

The Scheduler maintains per-instance queue state using one of two representations, determined by the active scheduling strategy.

### Probabilistic (CDF Representation)

Used by: `probabilistic`, `adaptive_bootstrap`

```
InstanceQueueProbabilistic {
    instance_id: string
    quantiles: float[]    # e.g., [0.5, 0.9, 0.95, 0.99]
    values: float[]       # corresponding time in ms
}
```

The quantile-value pairs represent a cumulative distribution function (CDF) of the remaining queue time. For example, `quantiles=[0.5, 0.9], values=[100, 250]` means a 50% chance the queue completes within 100ms and 90% chance within 250ms.

### ExpectError (Point Estimate)

Used by: `min_time`, `po2`, `serverless`

```
InstanceQueueExpectError {
    instance_id: string
    expected_time_ms: float    # expected remaining time
    error_margin_ms: float     # accumulated error (standard deviation)
}
```

A point estimate with error margin. The expected time is the sum of all predicted task times; the error margin accumulates via the square root of sum of squared individual errors.

## Scheduling Strategies

### Adaptive Bootstrap (Default)

**Strategy name:** `adaptive_bootstrap`
**Queue model:** Probabilistic
**Prediction type:** `quantile`

Auto-detects cold-start vs. warm models per platform. On each scheduling call, checks whether all instances have a trained quantile predictor model on disk.

- **All warm:** Delegates to the Probabilistic strategy.
- **Any cold:** Falls back to thread-safe round-robin selection with no prediction call.

```
function adaptive_schedule(model_id, metadata, instances):
    all_warm = all(has_trained_model(model_id, inst.platform_info) for inst in instances)
    if all_warm:
        return probabilistic_schedule(model_id, metadata, instances)
    else:
        idx = atomic_increment(rr_counter) % len(instances)
        return instances[idx]  # no prediction call
```

| Parameter | Value | Description |
|---|---|---|
| `target_quantile` | 0.9 (configurable) | Passed to inner probabilistic strategy |

### Probabilistic

**Strategy name:** `probabilistic`
**Queue model:** Probabilistic
**Prediction type:** `quantile`

Monte Carlo sampling strategy. For each instance, samples from both the prediction quantile distribution and the queue quantile distribution, then selects the instance that "wins" (has minimum total time) across the most samples.

```
function probabilistic_select(predictions, queue_info):
    N_SAMPLES = 10
    random_percentiles = random(N_SAMPLES)  # uniform [0, 1)

    for each instance i:
        pred_times[i]  = interpolate(random_percentiles, pred[i].quantiles, pred[i].values)
        queue_times[i] = interpolate(random_percentiles, queue[i].quantiles, queue[i].values)
        total_times[i] = pred_times[i] + queue_times[i]

    # For each sample, find which instance has minimum total time
    winners = argmin(total_times, axis=instances)  # shape: (N_SAMPLES,)

    # Return instance with most wins
    return instance_ids[argmax(bincount(winners))]
```

**Queue update (Monte Carlo addition):**

```
function update_queue_probabilistic(instance_id, prediction):
    N_SAMPLES = 1000
    random_percentiles = random(N_SAMPLES)

    queue_samples = interpolate(random_percentiles, queue.quantiles, queue.values)
    task_samples  = interpolate(random_percentiles, pred.quantiles, pred.values)
    total_samples = queue_samples + task_samples

    new_values = [percentile(total_samples, q * 100) for q in queue.quantiles]
    queue.values = new_values
```

| Parameter | Default | Description |
|---|---|---|
| `target_quantile` | 0.9 | Used for instance selection tie-breaking |
| Selection samples | 10 | Number of Monte Carlo samples for selection |
| Queue update samples | 1000 | Number of Monte Carlo samples for queue update |

### Minimum Expected Time

**Strategy name:** `min_time`
**Queue model:** ExpectError
**Prediction type:** `expect_error`

Greedy strategy that selects the instance minimizing total expected completion time.

```
function min_time_select(predictions, queue_info):
    for each instance i:
        total[i] = queue[i].expected_time_ms + queue[i].error_margin_ms + pred[i].predicted_time_ms
    return instance with min(total)
```

**Queue update (error accumulation):**

```
function update_queue_min_time(instance_id, prediction):
    queue.expected_time_ms += prediction.predicted_time_ms
    queue.error_margin_ms  = sqrt(queue.error_margin_ms^2 + prediction.error_margin_ms^2)
```

### Power of Two Choices

**Strategy name:** `po2`
**Queue model:** ExpectError
**Prediction type:** `expect_error`

Randomly selects two instances and picks the one with lower expected total time.

```
function po2_select(predictions, queue_info):
    inst1 = random_choice(predictions)
    inst2 = random_choice(predictions)

    time1 = queue[inst1].expected_time_ms + pred[inst1].predicted_time_ms
    time2 = queue[inst2].expected_time_ms + pred[inst2].predicted_time_ms

    return inst1 if time1 < time2 else inst2
```

**Queue update:** Same as Minimum Expected Time (error accumulation formula).

### Round Robin

**Strategy name:** `round_robin`
**Queue model:** Probabilistic (unused)
**Prediction type:** `expect_error`

Cycles through instances in order. Does not use predictions or queue state for scheduling decisions.

```
function round_robin_select(predictions, queue_info):
    selected = predictions[counter % len(predictions)]
    counter += 1
    return selected.instance_id
```

**Queue update:** No-op.

### Random

**Strategy name:** `random`
**Queue model:** Probabilistic (unused)
**Prediction type:** `expect_error`

Uniformly randomly selects an instance. Represents the worst case for probabilistic scheduling (no intelligence).

```
function random_select(predictions, queue_info):
    return random_choice(predictions).instance_id
```

**Queue update:** No-op.

### Serverless Optimized

**Strategy name:** `serverless`
**Queue model:** ExpectError
**Prediction type:** `expect_error`

Identical to Minimum Expected Time but designed for serverless environments where instance startup latency matters.

**Selection:** Same as `min_time`.
**Queue update:** Same as `min_time` (error accumulation formula).

## Strategy Comparison

| Aspect | Adaptive Bootstrap | Probabilistic | Min Time | Power of Two | Round Robin | Random | Serverless |
|---|---|---|---|---|---|---|---|
| Prediction type | quantile | quantile | expect_error | expect_error | expect_error | expect_error | expect_error |
| Queue model | probabilistic | probabilistic | expect_error | expect_error | *(unused)* | *(unused)* | expect_error |
| Uses predictions | warm only | yes | yes | yes | no | no | yes |
| Selection method | delegates | Monte Carlo (10 samples) | greedy min | random 2-pick | cyclic | random | greedy min |
| Queue update | delegates | Monte Carlo (1000 samples) | error accumulation | error accumulation | no-op | no-op | error accumulation |
| Cold-start handling | round-robin fallback | *(requires trained model)* | *(requires trained model)* | *(requires trained model)* | inherent | inherent | *(requires trained model)* |
| Best for | general use | heterogeneous instances | low-variance workloads | large clusters | no predictor | baseline | serverless |

## Queue Update Formulas

### ExpectError Update

When a task with prediction `(predicted_time_ms, error_margin_ms)` is added to an instance queue:

```
new_expected = current_expected + predicted_time_ms
new_error    = sqrt(current_error^2 + predicted_error^2)
```

When a task is removed (e.g., work stealing subtraction):

```
new_expected = max(0, current_expected - predicted_time_ms)
new_error    = current_error  # error margin unchanged on subtraction
```

### Probabilistic Update

**Addition (task scheduled to instance):**

```
N = 1000
percentiles = uniform_random(N)
queue_samples = interpolate(percentiles, queue.quantiles, queue.values)
task_samples  = interpolate(percentiles, pred.quantiles, pred.values)
total_samples = queue_samples + task_samples
new_values    = [percentile(total_samples, q * 100) for q in queue.quantiles]
```

**Subtraction (task stolen from instance):**

```
N = 1000
percentiles = uniform_random(N)
queue_samples = interpolate(percentiles, queue.quantiles, queue.values)
task_samples  = interpolate(percentiles, pred.quantiles, pred.values)
updated_samples = max(queue_samples - task_samples, 0)
new_values = [percentile(updated_samples, q * 100) for q in queue.quantiles]
```

## Work Stealing

When a new instance registers, the Scheduler performs background work stealing to balance load:

```
function redistribute_tasks_on_registration(new_instance_id):
    1. Find donor candidates: active instances with same model_id
    2. Count active tasks (pending + running) across all instances
    3. Calculate donors to select: min(ceil(active_tasks / total_instances), num_donors)
    4. Randomly select that many donors
    5. In parallel: call GET /task/fetch on each donor to steal 1 task
    6. For each stolen task:
       a. Subtract predicted time from donor's queue info
       b. Enqueue to new instance's WorkerQueueThread
       c. Add predicted time to new instance's queue info
       d. Update task's assigned_instance in registry
    7. Set new instance status: INITIALIZING → ACTIVE
```

The new instance starts in `INITIALIZING` status and transitions to `ACTIVE` after work stealing completes (whether tasks were stolen or not).

## Transparent Proxy Router

The catch-all proxy route `/{path}` enables the Scheduler to act as a transparent reverse proxy. Any HTTP request not matching a scheduler-internal route is forwarded to a backend instance through the full scheduling pipeline.

```
function proxy_handler(path, request):
    1. Generate task_id = "proxy-{random_hex}"
    2. List all active instances
    3. Schedule: select instance via scheduling algorithm
    4. Register a Future for this task_id
    5. Create QueuedTask with path, body, method, headers in metadata
    6. Enqueue to selected instance's WorkerQueueThread
    7. Await Future (with proxy_timeout, default 300s)
    8. Return backend's response (status code, body, headers)
```

The proxy is mounted AFTER all scheduler-internal routes (`/v1/health`, `/v1/instance/*`, `/v1/task/*`, `/v1/strategy/*`, `/v1/callback/*`) so those take priority.

## Worker Queue Architecture

### WorkerQueueManager

Central coordinator that manages all `WorkerQueueThread` instances:

- **register_worker(id, endpoint, model_id):** Creates and starts a new thread
- **deregister_worker(id):** Stops thread, returns pending tasks
- **enqueue_task(id, task):** Routes task to the correct thread
- **get_queue_depth(id):** Returns thread's current queue size
- **get_all_queue_depths():** Returns depths for all workers

### WorkerQueueThread

Per-instance dedicated thread with a FIFO task queue:

```
function process_loop():
    http_client = new HttpClient(timeout=worker_http_timeout)

    while not shutdown:
        task = queue.get(timeout=1s)     # blocks until task or timeout
        current_task = task
        start_time = now()

        # Determine URL from metadata
        path = task.metadata.get("path", "v1/completions")
        url = f"{worker_endpoint}/{path}"

        # Execute with retry logic
        for attempt in 1..max_retries:
            try:
                response = http_client.request(method, url, json=task.task_input)
                response.raise_for_status()
                callback(TaskResult(status="completed", result=response.json()))
                break
            except ConnectError, ReadError:
                if attempt < max_retries:
                    sleep(retry_delay * 2^attempt)  # exponential backoff
                else:
                    callback(TaskResult(status="failed", error=str(e)))
            except HTTPStatusError:
                callback(TaskResult(status="failed", result=error_body, http_status_code=code))
                break  # no retry for 4xx/5xx
            except TimeoutException:
                callback(TaskResult(status="failed", error="timeout"))
                break  # no retry for timeout

        current_task = null
```

**Key behaviors:**
- Each instance gets exactly one thread (sequential task execution per instance)
- Tasks execute in FIFO order within each thread
- Transient connection errors (ConnectError, ReadError) are retried with exponential backoff
- HTTP status errors (4xx, 5xx) and timeouts are not retried
- Results are delivered via a thread-safe callback that bridges to the async event loop
