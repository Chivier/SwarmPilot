## PlatformInfo

Identifies the hardware and software environment for prediction.

### Schema

```python
class PlatformInfo(BaseModel):
  software_name: str     # Software/framework name (e.g., "pytorch", "tensorflow")
                         # Set to "exp" when in experiment mode
  software_version: str  # Software version (e.g., "2.0.1")
                         # Set to "exp" when in experiment mode
  hardware_name: str     # Hardware identifier (e.g., "nvidia-a100", "cpu-x86")
                         # Set to "exp" when in experiment mode
```

### Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `software_name` | string | Yes | Software/framework name |
| `software_version` | string | Yes | Software version string |
| `hardware_name` | string | Yes | Hardware identifier |

### Example

```json
{
  "software_name": "pytorch",
  "software_version": "2.0.1",
  "hardware_name": "nvidia-a100"
}
```

### Special Values

For **experiment mode**, set all fields to `"exp"`:
```json
{
  "software_name": "exp",
  "software_version": "exp",
  "hardware_name": "exp"
}
```

## PredictionRequest

Request body for the `/predict` endpoint.

### Schema

```python
class PredictionRequest(BaseModel):
  model_id: str               # Unique identifier of the model/tool to predict for
  platform_info: PlatformInfo # Platform environment specification
  prediction_type: str        # Type of prediction: "expect_error" or "quantile"
  features: Dict[str, Any]    # Feature dictionary for prediction
                              # Special key: "exp_runtime" enables experiment mode
```

### Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `model_id` | string | Yes | Unique identifier of the model/tool |
| `platform_info` | PlatformInfo | Yes | Platform specification |
| `prediction_type` | string | Yes | Type of prediction (see below) |
| `features` | object | Yes | Feature dictionary for prediction |

### Prediction Types

| Value | Description |
|-------|-------------|
| `"expect_error"` | Returns expected runtime and error margin |
| `"quantile"` | Returns runtime distribution as quantiles |

### Features

The `features` object should contain all feature keys that were used during training. Feature names and types must match exactly.

**Special key**: `exp_runtime` - When present, activates experiment mode and bypasses the trained model.

### Example

```json
{
  "model_id": "image-classification-v1",
  "platform_info": {
    "software_name": "pytorch",
    "software_version": "2.0.1",
    "hardware_name": "nvidia-a100"
  },
  "prediction_type": "expect_error",
  "features": {
    "image_width": 1920,
    "image_height": 1080,
    "batch_size": 32
  }
}
```

## PredictionResponse

Response body from the `/predict` endpoint.

### Schema

```python
class PredictionResponse(BaseModel):
  model_id: str
  platform_info: PlatformInfo
  prediction_type: str
  result: Dict[str, Any]  # Content varies by prediction_type
```

### Fields

| Field | Type | Description |
|-------|------|-------------|
| `model_id` | string | Model identifier used for prediction |
| `platform_info` | PlatformInfo | Platform specification |
| `prediction_type` | string | Type of prediction performed |
| `result` | object | Prediction result (format varies by type) |

### Result Format by Prediction Type

#### For `"expect_error"` type:

```json
{
  "expected_runtime_ms": 125.5,
  "error_margin_ms": 10.2
}
```

| Field | Type | Description |
|-------|------|-------------|
| `expected_runtime_ms` | number | Mean/average predicted runtime in milliseconds |
| `error_margin_ms` | number | Error margin (typically ±1 standard deviation) |

#### For `"quantile"` type:

```json
{
  "quantiles": {
    "0.5": 120.0,
    "0.9": 150.0,
    "0.95": 180.0,
    "0.99": 250.0
  }
}
```

| Field | Type | Description |
|-------|------|-------------|
| `quantiles` | object | Map of quantile values to runtime predictions |

The quantiles returned depend on what was configured during training.

### Complete Examples

**Example (expect_error type)**:
```json
{
  "model_id": "image-classification-v1",
  "platform_info": {
    "software_name": "pytorch",
    "software_version": "2.0.1",
    "hardware_name": "nvidia-a100"
  },
  "prediction_type": "expect_error",
  "result": {
    "expected_runtime_ms": 125.5,
    "error_margin_ms": 10.2
  }
}
```

**Example (quantile type)**:
```json
{
  "model_id": "image-classification-v1",
  "platform_info": {
    "software_name": "pytorch",
    "software_version": "2.0.1",
    "hardware_name": "nvidia-a100"
  },
  "prediction_type": "quantile",
  "result": {
    "quantiles": {
      "0.5": 120.0,
      "0.9": 150.0,
      "0.95": 180.0,
      "0.99": 250.0
    }
  }
}
```

## TrainingRequest

Request body for the `/train` endpoint.

### Schema

```python
class TrainingRequest(BaseModel):
  model_id: str                             # Model identifier to train
  platform_info: PlatformInfo               # Platform specification
  prediction_type: str                      # "expect_error" or "quantile"
  features_list: List[Dict[str, Any]]       # List of training samples
  training_config: Optional[Dict[str, Any]] # Optional training configuration
```

### Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `model_id` | string | Yes | Unique identifier for the model |
| `platform_info` | PlatformInfo | Yes | Platform specification |
| `prediction_type` | string | Yes | Type of prediction model to train |
| `features_list` | array | Yes | List of training data samples |
| `training_config` | object | No | Optional training configuration |

### Features List Requirements

Each item in `features_list` must contain:
- All feature keys that will be used for prediction
- A `runtime_ms` field with the actual measured runtime value

**Example feature item**:
```json
{
  "image_width": 1920,
  "image_height": 1080,
  "batch_size": 32,
  "runtime_ms": 125.3
}
```

### Training Configuration

Optional configuration object to customize training behavior. Configuration options vary by `prediction_type`.

#### For `"quantile"` prediction type:

```json
{
  "quantiles": [0.5, 0.9, 0.95, 0.99]
}
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `quantiles` | array of numbers | [0.5, 0.9, 0.95, 0.99] | List of quantiles to predict |
| `hidden_layers` | array of ints | [64, 32] | Hidden size of MLP |
| `learning_rate` | float | 0.01 | learning rate of ML model |
| `epochs` | int | 500 | epoch of ML model training |

**Constraints**:
- All values must be between 0 and 1
- At least one quantile must be specified

#### For `"expect_error"` prediction type:

No configuration options currently supported (field is ignored).

### Complete Examples

**Example (expect_error type)**:
```json
{
  "model_id": "image-classification-v1",
  "platform_info": {
    "software_name": "pytorch",
    "software_version": "2.0.1",
    "hardware_name": "nvidia-a100"
  },
  "prediction_type": "expect_error",
  "features_list": [
    {
      "image_width": 1920,
      "image_height": 1080,
      "batch_size": 32,
      "runtime_ms": 125.3
    },
    {
      "image_width": 1280,
      "image_height": 720,
      "batch_size": 32,
      "runtime_ms": 85.7
    }
  ]
}
```

**Example (quantile type with custom quantiles)**:
```json
{
  "model_id": "object-detection-v2",
  "platform_info": {
    "software_name": "tensorflow",
    "software_version": "2.12.0",
    "hardware_name": "nvidia-v100"
  },
  "prediction_type": "quantile",
  "features_list": [
    {
      "image_width": 1920,
      "image_height": 1080,
      "num_objects": 10,
      "runtime_ms": 245.8
    },
    {
      "image_width": 1280,
      "image_height": 720,
      "num_objects": 5,
      "runtime_ms": 132.4
    }
  ],
  "training_config": {
    "quantiles": [0.25, 0.5, 0.75, 0.9, 0.95, 0.99]
  }
}
```

## TrainingResponse

Response body from the `/train` endpoint.

### Schema

```python
class TrainingResponse(BaseModel):
  status: str              # "success" or "error"
  message: str             # Human-readable status message
  model_key: str           # Internal key identifying the trained model
  samples_trained: int     # Number of training samples processed
```

### Fields

| Field | Type | Description |
|-------|------|-------------|
| `status` | string | Training status: `"success"` or `"error"` |
| `message` | string | Human-readable status message |
| `model_key` | string | Internal key identifying the trained model |
| `samples_trained` | integer | Number of training samples processed |

### Model Key Format

The model key combines all identification parameters:
```
{model_id}__{software_name}-{software_version}__{hardware_name}
```

**Example**: `image-classification-v1__pytorch-2.0.1__nvidia-a100`

### Example

```json
{
  "status": "success",
  "message": "Model trained successfully",
  "model_key": "image-classification-v1__pytorch-2.0.1__nvidia-a100",
  "samples_trained": 150
}
```

## ModelListResponse

Response body from the `/list` endpoint.

### Schema

```python
class ModelListResponse(BaseModel):
  models: List[Dict[str, Any]]  # List of available models
```

### Fields

| Field | Type | Description |
|-------|------|-------------|
| `models` | array | List of model metadata objects |

### Model Entry Schema

Each model entry contains:

| Field | Type | Description |
|-------|------|-------------|
| `model_id` | string | Model identifier |
| `platform_info` | PlatformInfo | Platform specification |
| `prediction_type` | string | Type of prediction the model provides |
| `samples_count` | integer | Number of training samples used |
| `last_trained` | string | ISO 8601 timestamp of last training |

### Example

```json
{
  "models": [
    {
      "model_id": "image-classification-v1",
      "platform_info": {
        "software_name": "pytorch",
        "software_version": "2.0.1",
        "hardware_name": "nvidia-a100"
      },
      "prediction_type": "expect_error",
      "samples_count": 150,
      "last_trained": "2025-10-31T10:30:00Z"
    },
    {
      "model_id": "object-detection-v2",
      "platform_info": {
        "software_name": "tensorflow",
        "software_version": "2.12.0",
        "hardware_name": "nvidia-v100"
      },
      "prediction_type": "quantile",
      "samples_count": 200,
      "last_trained": "2025-10-30T15:45:00Z"
    }
  ]
}
```
