# Introduction

The Predictor is a stateless HTTP microservice that trains and serves MLP-based runtime predictions. Given historical feature-to-runtime observations, it trains lightweight neural networks to predict execution times for ML inference workloads. It supports two prediction modes: point estimates with error margins (expect_error) and quantile regression for SLA-aware scheduling (quantile). The service also provides a library API for direct Python integration and a WebSocket endpoint for real-time streaming predictions.

## Architecture

```
                      ┌─────────────┐
                      │   Client    │
                      └──────┬──────┘
                             │ HTTP / WebSocket / Python
                      ┌──────▼──────┐
                      │  Predictor  │  POST /train    → train model
                      │  (FastAPI)  │  POST /predict  → get prediction
                      └──┬──────┬───┘  WS /ws/predict → streaming
                         │      │
             ┌───────────┘      └───────────┐
             │                              │
      ┌──────▼──────┐               ┌──────▼──────┐
      │   Storage   │               │    Cache     │
      │  (joblib)   │               │  (in-memory) │
      └─────────────┘               └──────────────┘
```

- **Predictor** exposes training, prediction, and model management endpoints.
- **Storage** persists trained models as joblib files on disk.
- **Cache** holds recently loaded models in memory for fast inference (LRU, default 100 entries).

## Key Concepts

| Concept | Meaning |
|---|---|
| **model_id** | User-defined identifier for the model being predicted (e.g., `"llama-7b"`) |
| **platform_info** | Execution environment: `software_name`, `software_version`, `hardware_name` |
| **prediction_type** | Either `"expect_error"` (point estimate + margin) or `"quantile"` (percentile distribution) |
| **model_key** | Storage key derived from `{model_id}__{software_name}-{software_version}__{hardware_name}` |
| **features** | Numeric input features (e.g., `batch_size`, `sequence_length`) used for prediction |
| **runtime_ms** | Target variable: measured execution time in milliseconds |
| **preprocessor** | Transform applied to features before training/prediction (e.g., multiply, remove) |

## Prediction Modes

### Expect Error (Point Estimate)

The `expect_error` mode trains an MLP with MSE loss to predict the expected runtime. After training, it computes an error margin from the mean absolute residuals. This mode is suitable when a single best-guess value with a confidence bound is needed.

Output: `expected_runtime_ms` and `error_margin_ms`.

### Quantile Regression

The `quantile` mode trains an MLP with pinball loss and a base+delta architecture to predict multiple quantile levels simultaneously (default: 50th, 90th, 95th, 99th percentiles). Monotonicity between quantiles is guaranteed by construction. This mode is suitable for SLA-aware scheduling where different confidence levels are needed.

Output: a dictionary of quantile levels to predicted runtimes (e.g., `{"0.5": 120, "0.9": 150, "0.95": 180, "0.99": 250}`).

### Experiment Mode

When features contain an `exp_runtime` field or all `platform_info` fields are set to `"exp"`, the service generates synthetic predictions without requiring a trained model. This mode is used for testing and integration without real training data.
