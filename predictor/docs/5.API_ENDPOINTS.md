## POST `/predict`

Get runtime prediction for a task on a specific platform.

### Request

**Method**: `POST`
**Path**: `/predict`
**Content-Type**: `application/json`

**Body**: [PredictionRequest](4.DATA_MODELS.md#predictionrequest)

### Response

**Content-Type**: `application/json`

**Body**: [PredictionResponse](4.DATA_MODELS.md#predictionresponse)

### Status Codes

| Code | Description |
|------|-------------|
| `200 OK` | Prediction successful |
| `400 Bad Request` | Invalid request parameters (missing fields, invalid types, etc.) |
| `404 Not Found` | No trained model found for the specified model_id and platform_info |
| `500 Internal Server Error` | Prediction failed due to internal error |

### Examples

#### Example 1: Expect & Error Prediction

**Request**:
```bash
curl -X POST http://localhost:8000/predict \
  -H "Content-Type: application/json" \
  -d '{
    "model_id": "image-classification-v1",
    "platform_info": {
      "software_name": "pytorch",
      "software_version": "2.0.1",
      "hardware_name": "nvidia-a100"
    },
    "prediction_type": "expect_error",
    "features": {
      "image_width": 1920,
      "image_height": 1080,
      "batch_size": 32
    }
  }'
```

**Response** (200 OK):
```json
{
  "model_id": "image-classification-v1",
  "platform_info": {
    "software_name": "pytorch",
    "software_version": "2.0.1",
    "hardware_name": "nvidia-a100"
  },
  "prediction_type": "expect_error",
  "result": {
    "expected_runtime_ms": 125.5,
    "error_margin_ms": 10.2
  }
}
```

#### Example 2: Quantile Prediction

**Request**:
```bash
curl -X POST http://localhost:8000/predict \
  -H "Content-Type: application/json" \
  -d '{
    "model_id": "image-classification-v1",
    "platform_info": {
      "software_name": "pytorch",
      "software_version": "2.0.1",
      "hardware_name": "nvidia-a100"
    },
    "prediction_type": "quantile",
    "features": {
      "image_width": 1920,
      "image_height": 1080,
      "batch_size": 32
    }
  }'
```

**Response** (200 OK):
```json
{
  "model_id": "image-classification-v1",
  "platform_info": {
    "software_name": "pytorch",
    "software_version": "2.0.1",
    "hardware_name": "nvidia-a100"
  },
  "prediction_type": "quantile",
  "result": {
    "quantiles": {
      "0.5": 120.0,
      "0.9": 150.0,
      "0.95": 180.0,
      "0.99": 250.0
    }
  }
}
```

#### Example 3: Model Not Found Error

**Request**:
```bash
curl -X POST http://localhost:8000/predict \
  -H "Content-Type: application/json" \
  -d '{
    "model_id": "non-existent-model",
    "platform_info": {
      "software_name": "pytorch",
      "software_version": "2.0.1",
      "hardware_name": "nvidia-a100"
    },
    "prediction_type": "expect_error",
    "features": {
      "image_width": 1920
    }
  }'
```

**Response** (404 Not Found):
```json
{
  "error": "Model not found",
  "model_id": "non-existent-model",
  "platform_info": {
    "software_name": "pytorch",
    "software_version": "2.0.1",
    "hardware_name": "nvidia-a100"
  }
}
```

## POST `/train`

Train or update a prediction model with new data.

### Request

**Method**: `POST`
**Path**: `/train`
**Content-Type**: `application/json`

**Body**: [TrainingRequest](4.DATA_MODELS.md#trainingrequest)

### Response

**Content-Type**: `application/json`

**Body**: [TrainingResponse](4.DATA_MODELS.md#trainingresponse)

### Status Codes

| Code | Description |
|------|-------------|
| `200 OK` | Training successful |
| `400 Bad Request` | Invalid training data (insufficient samples, invalid features, etc.) |
| `500 Internal Server Error` | Training failed due to internal error |

### Examples

#### Example 1: Train Expect & Error Model

**Request**:
```bash
curl -X POST http://localhost:8000/train \
  -H "Content-Type: application/json" \
  -d '{
    "model_id": "image-classification-v1",
    "platform_info": {
      "software_name": "pytorch",
      "software_version": "2.0.1",
      "hardware_name": "nvidia-a100"
    },
    "prediction_type": "expect_error",
    "features_list": [
      {
        "image_width": 1920,
        "image_height": 1080,
        "batch_size": 32,
        "runtime_ms": 125.3
      },
      {
        "image_width": 1280,
        "image_height": 720,
        "batch_size": 32,
        "runtime_ms": 85.7
      },
      {
        "image_width": 3840,
        "image_height": 2160,
        "batch_size": 16,
        "runtime_ms": 245.1
      }
    ]
  }'
```

**Response** (200 OK):
```json
{
  "status": "success",
  "message": "Model trained successfully",
  "model_key": "image-classification-v1__pytorch-2.0.1__nvidia-a100",
  "samples_trained": 3
}
```

#### Example 2: Train Quantile Model with Custom Configuration

**Request**:
```bash
curl -X POST http://localhost:8000/train \
  -H "Content-Type: application/json" \
  -d '{
    "model_id": "object-detection-v2",
    "platform_info": {
      "software_name": "tensorflow",
      "software_version": "2.12.0",
      "hardware_name": "nvidia-v100"
    },
    "prediction_type": "quantile",
    "features_list": [
      {
        "image_width": 1920,
        "image_height": 1080,
        "num_objects": 10,
        "runtime_ms": 245.8
      },
      {
        "image_width": 1280,
        "image_height": 720,
        "num_objects": 5,
        "runtime_ms": 132.4
      }
    ],
    "training_config": {
      "quantiles": [0.5, 0.9, 0.95, 0.99]
    }
  }'
```

**Response** (200 OK):
```json
{
  "status": "success",
  "message": "Model trained successfully",
  "model_key": "object-detection-v2__tensorflow-2.12.0__nvidia-v100",
  "samples_trained": 2
}
```

#### Example 3: Invalid Training Data Error

**Request**:
```bash
curl -X POST http://localhost:8000/train \
  -H "Content-Type: application/json" \
  -d '{
    "model_id": "test-model",
    "platform_info": {
      "software_name": "pytorch",
      "software_version": "2.0.1",
      "hardware_name": "nvidia-a100"
    },
    "prediction_type": "expect_error",
    "features_list": [
      {
        "image_width": 1920,
        "runtime_ms": 125.3
      }
    ]
  }'
```

**Response** (400 Bad Request):
```json
{
  "error": "Invalid training data",
  "message": "Insufficient training samples. Minimum 10 samples required.",
  "samples_provided": 1
}
```

### Notes

- The `training_config` field is optional. If omitted for quantile models, default quantiles `[0.5, 0.9, 0.95, 0.99]` are used
- For `expect_error` prediction type, `training_config` is ignored
- The same model key can be retrained with updated data to improve accuracy
- Minimum 10 training samples recommended (50+ for production use)

## GET `/list`

List all available trained models.

### Request

**Method**: `GET`
**Path**: `/list`

No request body required.

### Response

**Content-Type**: `application/json`

**Body**: [ModelListResponse](4.DATA_MODELS.md#modellistresponse)

### Status Codes

| Code | Description |
|------|-------------|
| `200 OK` | List retrieved successfully |
| `500 Internal Server Error` | Failed to retrieve models |

### Example

**Request**:
```bash
curl -X GET http://localhost:8000/list
```

**Response** (200 OK):
```json
{
  "models": [
    {
      "model_id": "image-classification-v1",
      "platform_info": {
        "software_name": "pytorch",
        "software_version": "2.0.1",
        "hardware_name": "nvidia-a100"
      },
      "prediction_type": "expect_error",
      "samples_count": 150,
      "last_trained": "2025-10-31T10:30:00Z"
    },
    {
      "model_id": "object-detection-v2",
      "platform_info": {
        "software_name": "tensorflow",
        "software_version": "2.12.0",
        "hardware_name": "nvidia-v100"
      },
      "prediction_type": "quantile",
      "samples_count": 200,
      "last_trained": "2025-10-30T15:45:00Z"
    }
  ]
}
```

### Empty Response

When no models are trained:
```json
{
  "models": []
}
```

## GET `/health`

Check the health status of the predictor service.

### Request

**Method**: `GET`
**Path**: `/health`

No request body required.

### Response

**Content-Type**: `application/json`

### Status Codes

| Code | Description |
|------|-------------|
| `200 OK` | Service is healthy and operational |
| `503 Service Unavailable` | Service is unhealthy or starting up |

### Examples

#### Healthy Service

**Request**:
```bash
curl -X GET http://localhost:8000/health
```

**Response** (200 OK):
```json
{
  "status": "healthy"
}
```

#### Unhealthy Service

**Response** (503 Service Unavailable):
```json
{
  "status": "unhealthy",
  "reason": "Model storage unavailable"
}
```

## Common Request Headers

All endpoints accept the following standard headers:

| Header | Value | Required |
|--------|-------|----------|
| `Content-Type` | `application/json` | Yes (for POST) |
| `Accept` | `application/json` | No (default) |

## Rate Limiting

Currently, no rate limiting is implemented. The service is designed for internal use within the swarmpilot system.

## Authentication

No authentication is required. The service is designed to run as an internal component.

## Base URL

The default base URL is:
```
http://localhost:8000
```

In production, this should be configured to point to the actual service address.

## Error Response Format

All error responses follow this general structure:

```json
{
  "error": "Error category",
  "message": "Detailed error message",
  "additional_field": "Additional context"
}
```

For more details on error handling, see [Error Handling Guide](7.ERROR_HANDLING.md).
