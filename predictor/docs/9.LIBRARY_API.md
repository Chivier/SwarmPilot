## Quick Start

```python
from src.api import PredictorCore
from src.models import PlatformInfo

# Initialize the high-level API
core = PredictorCore()

# Define platform information
platform = PlatformInfo(
    software_name="PyTorch",
    software_version="2.0",
    hardware_name="NVIDIA A100"
)

# 1. Collect training samples (minimum 10 required)
for i in range(15):
    core.collect(
        model_id="my_model",
        platform_info=platform,
        prediction_type="quantile",
        features={"batch_size": 32 + i * 4, "image_size": 224},
        runtime_ms=100.0 + i * 15.0,
    )

# 2. Train on accumulated data
result = core.train(
    model_id="my_model",
    platform_info=platform,
    prediction_type="quantile",
)
print(f"Trained with {result.samples_trained} samples")

# 3. Make predictions
prediction = core.predict(
    model_id="my_model",
    platform_info=platform,
    prediction_type="quantile",
    features={"batch_size": 48, "image_size": 224},
)
print(f"Prediction: {prediction.result}")
```

## Overview

The predictor library API provides a **two-level architecture**:

| Level | Class | Purpose | When to Use |
|-------|-------|---------|-------------|
| **High-Level** | `PredictorCore` | Simple prediction, training via accumulator pattern | Most use cases |
| **Low-Level** | `PredictorLowLevel` | Full control over model management, storage, cache | Advanced control |

**Key Design Decisions:**

| Feature | Decision |
|---------|----------|
| **Sync/Async** | `predict()` is synchronous (fast), `train()` has both sync and async variants |
| **Caching** | Built-in (transparent at high-level, controllable at low-level) |
| **Training Data** | Accumulator pattern - `collect()` data over time, then `train()` |
| **Feature Schema** | First training batch defines expected features; prediction auto-filters extras and warns on missing |
| **Minimum Samples** | 10 samples required for training |

## High-Level API (`PredictorCore`)

The high-level API wraps `PredictorLowLevel` to provide a simple interface using the accumulator pattern.

### Constructor

```python
class PredictorCore:
    def __init__(
        self,
        low_level: PredictorLowLevel | None = None,
        storage_dir: str | None = None,
        max_workers: int = 4,
    ) -> None:
        """Initialize high-level predictor API.

        Args:
            low_level: Low-level API instance. Creates new if None.
            storage_dir: Override storage directory (only if low_level is None).
            max_workers: Max workers for async operations.
        """
```

### Accumulator Pattern Methods

The accumulator pattern separates data collection from training:

#### `collect()`

```python
def collect(
    self,
    model_id: str,
    platform_info: PlatformInfo,
    prediction_type: str,
    features: dict[str, Any],
    runtime_ms: float,
) -> None:
    """Add a sample to the accumulator.

    The first sample collected defines the feature schema.
    Subsequent samples must have the same features.

    Args:
        model_id: Model identifier.
        platform_info: Platform information.
        prediction_type: Type of prediction ("expect_error", "quantile",
                        "linear_regression", "decision_tree").
        features: Feature dictionary (without runtime_ms).
        runtime_ms: Measured runtime in milliseconds.

    Raises:
        ValidationError: If features are inconsistent with schema.
    """
```


```python
def clear_collected(

    model_id: str,
```

### Training Methods

#### `train()`

```python
def train(
    self,
    model_id: str,
    platform_info: PlatformInfo,
    prediction_type: str,
    config: dict[str, Any] | None = None,
) -> TrainingResult:
    """Train on accumulated data.

    Args:
        model_id: Model identifier.
        platform_info: Platform information.
        prediction_type: Type of prediction.
        config: Optional training configuration.

            - quantiles: List of quantile values for quantile predictor
            - learning_rate: Learning rate for neural network predictors

    Returns:
        TrainingResult with training metadata.

    Raises:
        ValidationError: If no data collected or < 10 samples.
        TrainingError: If training fails.
    """
```

### Error Handling Examples

```python
from src.api import (
    PredictorCore,
    ModelNotFoundError,
    ValidationError,
    TrainingError,
    PredictionError,
)

core = PredictorCore()

# Handle missing model
try:
    prediction = core.predict(
        model_id="nonexistent",
        platform_info=platform,
        prediction_type="quantile",
        features={"batch_size": 32},
    )
except ModelNotFoundError as e:
    print(f"Model not found: {e}")

# Handle missing features
try:
    prediction = core.predict(
        model_id="my_model",
        platform_info=platform,
        prediction_type="quantile",
        features={"batch_size": 32},  # Missing "image_size"
    )
except ValidationError as e:
    print(f"Invalid input: {e}")
    # Output: "Missing required features: ['image_size']..."

# Handle insufficient training data
try:
    for i in range(5):  # Only 5 samples (need 10)
        core.collect("model", platform, "quantile", {"x": i}, i * 10)
    core.train("model", platform, "quantile")
except ValidationError as e:
    print(f"Validation error: {e}")
    # Output: "Insufficient training data. Got 5 samples, but minimum 10..."
```

## Feature Schema Handling

### How Feature Schema Works

1. **First sample defines schema**: When you first call `collect()`, the feature names are extracted
2. **Schema is enforced**: Subsequent `collect()` calls must have exactly the same features
3. **Schema persists with model**: After training, the schema is saved with the model

### Prediction Behavior

**Auto-filter extra features:**

```python
# Model trained with: ["batch_size", "image_size"]
prediction = core.predict(
    model_id="my_model",
    platform_info=platform,
    prediction_type="quantile",
    features={
        "batch_size": 32,
        "image_size": 224,
        "extra_feature": 999,  # Automatically filtered out (ignored)
    },
)
# Works fine - extra features are silently ignored
```

**Error on missing features:**

```python
# Model trained with: ["batch_size", "image_size"]
try:
    prediction = core.predict(
        model_id="my_model",
        platform_info=platform,
        prediction_type="quantile",
        features={"batch_size": 32},  # Missing "image_size"!
    )
except ValidationError as e:
    print(e)
    # Output: "Missing required features: ['image_size'].
    #          Model expects: ['batch_size', 'image_size']"
```

### Schema Consistency During Collection

```python
# First sample defines schema
core.collect("model", platform, "quantile", {"a": 1, "b": 2}, 100.0)

# Second sample must have same features
core.collect("model", platform, "quantile", {"a": 5, "b": 6}, 150.0)  # OK

# Different features will raise error
try:
    core.collect("model", platform, "quantile", {"a": 5, "c": 6}, 150.0)
except ValidationError as e:
    print(e)
    # Output: "Sample has inconsistent features.
    #          Expected: ['a', 'b'], Missing: ['b'], Extra: ['c']"
```

## Complete Usage Examples

### High-Level API: Full Workflow

```python
from src.api import PredictorCore, ValidationError, ModelNotFoundError
from src.models import PlatformInfo

# Initialize
core = PredictorCore()

platform = PlatformInfo(
    software_name="PyTorch",
    software_version="2.0",
    hardware_name="NVIDIA A100"
)

# ==================
# COLLECT SAMPLES
# ==================
# Simulate collecting runtime measurements during actual execution
training_data = [
    {"batch_size": 32, "image_size": 224, "runtime_ms": 145.3},
    {"batch_size": 64, "image_size": 224, "runtime_ms": 278.1},
    {"batch_size": 32, "image_size": 448, "runtime_ms": 520.4},
    {"batch_size": 64, "image_size": 448, "runtime_ms": 1024.7},
    # ... more samples ...
]

for sample in training_data:
    core.collect(
        model_id="inference_model",
        platform_info=platform,
        prediction_type="quantile",
        features={
            "batch_size": sample["batch_size"],
            "image_size": sample["image_size"],
        },
        runtime_ms=sample["runtime_ms"],
    )

# Check how many samples collected
count = core.get_collected_count("inference_model", platform, "quantile")
print(f"Collected {count} samples")

# ==================
# TRAIN
# ==================
# Train on accumulated data (clears accumulator after success)
result = core.train(
    model_id="inference_model",
    platform_info=platform,
    prediction_type="quantile",
    config={
        "epochs": 500,
        "quantiles": [0.5, 0.9, 0.99],
    },
)
print(f"Training: {result.message}")

# ==================
# PREDICT
# ==================
# Single prediction
prediction = core.predict(
    model_id="inference_model",
    platform_info=platform,
    prediction_type="quantile",
    features={"batch_size": 48, "image_size": 336},
)
print(f"Predicted quantiles: {prediction.result}")
# Output: {"quantiles": {0.5: 350.0, 0.9: 420.0, 0.99: 480.0}}

# Batch prediction
predictions = core.batch_predict(
    model_id="inference_model",
    platform_info=platform,
    prediction_type="quantile",
    features_list=[
        {"batch_size": 32, "image_size": 224},
        {"batch_size": 64, "image_size": 224},
        {"batch_size": 128, "image_size": 224},
    ],
)
for pred in predictions:
    print(f"Prediction: {pred.result}")
```

### Low-Level API: Direct Control

```python
from src.api import PredictorLowLevel, ModelNotFoundError
from src.models import PlatformInfo

# Initialize
low = PredictorLowLevel(cache_size=50)

platform = PlatformInfo(
    software_name="PyTorch",
    software_version="2.0",
    hardware_name="NVIDIA A100"
)

# ==================
# DIRECT TRAINING
# ==================
# Training data with runtime_ms included
training_data = [
    {"batch_size": 32, "image_size": 224, "runtime_ms": 145.3},
    {"batch_size": 64, "image_size": 224, "runtime_ms": 278.1},
    # ... at least 10 samples ...
]

# Train predictor (not saved yet)
predictor = low.train_predictor(
    features_list=training_data,
    prediction_type="expect_error",
    config={"epochs": 300},
)

# Save to storage
low.save_model("my_model", platform, "expect_error", predictor)

# ==================
# LOAD AND PREDICT
# ==================
# Load from storage (uses cache if available)
loaded_predictor = low.load_model("my_model", platform, "expect_error")

# Predict directly
result = low.predict_with_predictor(
    loaded_predictor,
    {"batch_size": 48, "image_size": 224}
)
print(f"Prediction: {result}")

# ==================
# MODEL MANAGEMENT
# ==================
# List all models
all_models = low.list_models()
print(f"Total models: {len(all_models)}")

# Filter by prediction type
quantile_models = low.list_models(prediction_type="quantile")

# Get model info
info = low.get_model_info("my_model", platform, "expect_error")
if info:
    print(f"Model trained with {info.samples_count} samples at {info.last_trained}")
    print(f"Expected features: {info.feature_names}")

# Check existence
exists = low.model_exists("my_model", platform, "expect_error")
print(f"Model exists: {exists}")

# Delete model
deleted = low.delete_model("my_model", platform, "expect_error")
print(f"Model deleted: {deleted}")

# ==================
# CACHE CONTROL
# ==================
# Get cache statistics
stats = low.get_cache_stats()
print(f"Cache size: {stats['size']}/{stats['max_size']}")
print(f"Hit rate: {stats['hit_rate_percent']:.1f}%")

# Invalidate specific model
low.invalidate_cache("my_model", platform, "expect_error")

# Clear entire cache
low.clear_cache()

# ==================
# STORAGE INFO
# ==================
storage_info = low.get_storage_info()
print(f"Storage path: {storage_info}")

# Generate model key (for debugging)
key = low.generate_model_key("my_model", platform, "expect_error")
print(f"Model key: {key}")
# Output: "my_model__PyTorch-2.0__NVIDIA A100__expect_error"
```

### Async Training

```python
import asyncio
from src.api import PredictorCore
from src.models import PlatformInfo

async def train_model():
    core = PredictorCore()
    platform = PlatformInfo(
        software_name="PyTorch",
        software_version="2.0",
        hardware_name="NVIDIA A100"
    )

    # Collect samples
    for i in range(15):
        core.collect(
            model_id="async_model",
            platform_info=platform,
            prediction_type="quantile",
            features={"x": i, "y": i * 2},
            runtime_ms=100.0 + i * 10,
        )

    # Train asynchronously (non-blocking)
    result = await core.train_async(
        model_id="async_model",
        platform_info=platform,
        prediction_type="quantile",
    )
    print(f"Training completed: {result.message}")

# Run async
asyncio.run(train_model())
```

## Thread Safety

The library API is designed for concurrent use:

| Component | Thread Safety |
|-----------|--------------|
| `PredictorLowLevel` | Uses `threading.RLock` for cache/storage operations |
| `PredictorCore` | Uses lock for accumulated samples dictionary |
| `ModelCache` | Thread-safe internally |
| Async Operations | Uses `ThreadPoolExecutor` |

**Safe for:**
- Concurrent predictions from multiple threads
- Multiple `collect()` calls from different threads
- Simultaneous read operations

**Not recommended:**
- Training the same model from multiple threads simultaneously
