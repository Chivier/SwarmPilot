## Quick Start

### 1. Check Service Health

First, verify the service is running:

```bash
curl -X GET http://localhost:8000/health
```

Expected response:
```json
{
  "status": "healthy"
}
```

### 2. Train Your First Model

```bash
curl -X POST http://localhost:8000/train \
  -H "Content-Type: application/json" \
  -d '{
    "model_id": "my-first-model",
    "platform_info": {
      "software_name": "pytorch",
      "software_version": "2.0.1",
      "hardware_name": "nvidia-a100"
    },
    "prediction_type": "expect_error",
    "features_list": [
      {"input_size": 1024, "batch_size": 16, "runtime_ms": 45.2},
      {"input_size": 2048, "batch_size": 16, "runtime_ms": 89.5},
      {"input_size": 1024, "batch_size": 32, "runtime_ms": 42.1},
      {"input_size": 2048, "batch_size": 32, "runtime_ms": 85.3},
      {"input_size": 4096, "batch_size": 16, "runtime_ms": 178.7},
      {"input_size": 4096, "batch_size": 32, "runtime_ms": 172.4}
    ]
  }'
```

### 3. Get a Prediction

```bash
curl -X POST http://localhost:8000/predict \
  -H "Content-Type: application/json" \
  -d '{
    "model_id": "my-first-model",
    "platform_info": {
      "software_name": "pytorch",
      "software_version": "2.0.1",
      "hardware_name": "nvidia-a100"
    },
    "prediction_type": "expect_error",
    "features": {
      "input_size": 1536,
      "batch_size": 24
    }
  }'
```

## Experiment Mode

Experiment mode allows you to test without trained models. The predictor will return synthetic predictions based on `exp_runtime`.

### Method 1: Using exp_runtime in Features

```bash
curl -X POST http://localhost:8000/predict \
  -H "Content-Type: application/json" \
  -d '{
    "model_id": "test-model",
    "platform_info": {
      "software_name": "pytorch",
      "software_version": "2.0.1",
      "hardware_name": "nvidia-a100"
    },
    "prediction_type": "expect_error",
    "features": {
      "exp_runtime": 100.0,
      "any_other_feature": "value"
    }
  }'
```

Response:
```json
{
  "model_id": "test-model",
  "platform_info": {
    "software_name": "pytorch",
    "software_version": "2.0.1",
    "hardware_name": "nvidia-a100"
  },
  "prediction_type": "expect_error",
  "result": {
    "expected_runtime_ms": 100.0,
    "error_margin_ms": 5.0
  }
}
```

### Method 2: Using "exp" Platform Info

```bash
curl -X POST http://localhost:8000/predict \
  -H "Content-Type: application/json" \
  -d '{
    "model_id": "test-model",
    "platform_info": {
      "software_name": "exp",
      "software_version": "exp",
      "hardware_name": "exp"
    },
    "prediction_type": "quantile",
    "features": {
      "exp_runtime": 150.0
    }
  }'
```

Response:
```json
{
  "model_id": "test-model",
  "platform_info": {
    "software_name": "exp",
    "software_version": "exp",
    "hardware_name": "exp"
  },
  "prediction_type": "quantile",
  "result": {
    "quantiles": {
      "0.5": 150.0,
      "0.9": 157.5,

```

## Advanced Usage

### Incremental Training

Update an existing model with new data:

```bash
# Initial training
curl -X POST http://localhost:8000/train \
  -H "Content-Type: application/json" \
  -d '{
    "model_id": "video-encoder",
    "platform_info": {
      "software_name": "ffmpeg",
      "software_version": "5.1.0",
      "hardware_name": "nvidia-a100"
    },
    "prediction_type": "expect_error",
    "features_list": [
      {"resolution": 1080, "fps": 30, "duration_sec": 60, "runtime_ms": 5200},
      {"resolution": 720, "fps": 30, "duration_sec": 60, "runtime_ms": 3100}
    ]
  }'

# Later: add more training data
curl -X POST http://localhost:8000/train \
  -H "Content-Type: application/json" \
  -d '{
    "model_id": "video-encoder",
    "platform_info": {
      "software_name": "ffmpeg",
      "software_version": "5.1.0",
      "hardware_name": "nvidia-a100"
    },
    "prediction_type": "expect_error",
    "features_list": [
      {"resolution": 1080, "fps": 60, "duration_sec": 60, "runtime_ms": 8500},
      {"resolution": 4320, "fps": 30, "duration_sec": 60, "runtime_ms": 18200}
    ]
  }'
```

```bash
curl -X GET http://localhost:8000/list | jq '.models[] | select(.platform_info.hardware_name == "nvidia-a100")'
```

### Using with Python

```python
import requests

PREDICTOR_URL = "http://localhost:8000"

# Train a model
training_data = {
    "model_id": "my-model",
    "platform_info": {
        "software_name": "pytorch",
        "software_version": "2.0.1",
        "hardware_name": "nvidia-a100"
    },
    "prediction_type": "expect_error",
    "features_list": [
        {"input_size": 1024, "batch_size": 16, "runtime_ms": 45.2},
        {"input_size": 2048, "batch_size": 16, "runtime_ms": 89.5},
    ]
}

response = requests.post(f"{PREDICTOR_URL}/train", json=training_data)
print(f"Training result: {response.json()}")

# Get prediction
prediction_request = {
    "model_id": "my-model",
    "platform_info": {
        "software_name": "pytorch",
        "software_version": "2.0.1",
        "hardware_name": "nvidia-a100"
    },
    "prediction_type": "expect_error",
    "features": {
        "input_size": 1536,
        "batch_size": 24
    }
}

response = requests.post(f"{PREDICTOR_URL}/predict", json=prediction_request)
result = response.json()
print(f"Predicted runtime: {result['result']['expected_runtime_ms']}ms")
```

## Tips and Best Practices

1. **Always collect sufficient training data**: Minimum 50 samples for production models
2. **Use consistent feature names**: Feature names are case-sensitive
3. **Include all relevant features**: Missing features will cause prediction failures
4. **Test with experiment mode first**: Validate your integration before training models
5. **Monitor prediction accuracy**: Compare predictions with actual runtimes and retrain as needed
6. **Use appropriate prediction types**: Choose based on your scheduling requirements
