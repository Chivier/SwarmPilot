## Common Error Scenarios

### 1. Model Not Found (404)

**Scenario**: Attempting to get predictions for a model that hasn't been trained.

#### Error Response

```json
{
  "error": "Model not found",
  "model_id": "my-model",
  "platform_info": {
    "software_name": "pytorch",
    "software_version": "2.0.1",
    "hardware_name": "nvidia-a100"
  }
}
```

#### Causes

- Model was never trained for this combination of `model_id` and `platform_info`
- Typo in `model_id`, `software_name`, `software_version`, or `hardware_name`
- Model was trained on a different platform configuration

#### Solutions

1. **Verify the model exists**:
   ```bash
   curl -X GET http://localhost:8000/list | jq '.models[] | select(.model_id == "my-model")'
   ```

2. **Check exact platform info**:
   - Ensure `software_name`, `software_version`, and `hardware_name` match exactly (case-sensitive)
   - Verify there are no extra spaces or special characters

3. **Train the model first**:
   ```bash
   curl -X POST http://localhost:8000/train \
     -H "Content-Type: application/json" \
     -d '{
       "model_id": "my-model",
       "platform_info": {...},
       "prediction_type": "expect_error",
       "features_list": [...]
     }'
   ```

4. **Use experiment mode for testing**:
   ```bash
   # Add exp_runtime to features to bypass model requirement
   curl -X POST http://localhost:8000/predict \
     -H "Content-Type: application/json" \
     -d '{
       "model_id": "my-model",
       "platform_info": {...},
       "prediction_type": "expect_error",
       "features": {
         "exp_runtime": 100.0,
         "other_features": "..."
       }
     }'
   ```

### 2. Invalid Features (400)

**Scenario**: Features in prediction request don't match training features.

#### Error Response

```json
{
  "error": "Invalid features",
  "message": "Missing required feature: 'batch_size'",
  "expected_features": ["image_width", "image_height", "batch_size"],
  "provided_features": ["image_width", "image_height"]
}
```

#### Causes

- Missing features that were present during training
- Feature name typo (case-sensitive)
- Extra features that weren't in training data
- Feature type mismatch (string vs number)

#### Solutions

1. **Check expected features**:
   ```bash
   # List models to see what features they expect
   curl -X GET http://localhost:8000/list
   ```

2. **Verify feature names match exactly**:
   ```python
   # Training
   training_features = ["image_width", "image_height", "batch_size"]

   # Prediction - must match exactly
   prediction_features = {
       "image_width": 1920,
       "image_height": 1080,
       "batch_size": 32  # Don't forget this!
   }
   ```

3. **Check feature types**:
   ```python
   # Wrong - string instead of number
   {"batch_size": "32"}

   # Correct
   {"batch_size": 32}
   ```

4. **Retrain if feature set changed**:
   ```bash
   # If you need different features, retrain the model
   curl -X POST http://localhost:8000/train \
     -H "Content-Type: application/json" \
     -d '{
       "model_id": "my-model",
       "platform_info": {...},
       "features_list": [
         {"new_feature_set": "with all required features", "runtime_ms": 123}
       ]
     }'
   ```

### 3. Insufficient Training Data (400)

**Scenario**: Not enough training samples provided.

#### Error Response

```json
{
  "error": "Invalid training data",
  "message": "Insufficient training samples. Minimum 10 samples required.",
  "samples_provided": 5
}
```

#### Causes

- Less than 10 training samples provided
- Poor model quality with too few samples

#### Solutions

1. **Collect more training data**:
   ```bash
   # Minimum 10 samples, recommended 50+
   curl -X POST http://localhost:8000/train \
     -H "Content-Type: application/json" \
     -d '{
       "features_list": [
         {"feature1": 1, "runtime_ms": 10},
         {"feature1": 2, "runtime_ms": 20},
         # ... at least 10 total samples
       ]
     }'
   ```

2. **Use experiment mode while collecting data**:
   ```bash
   # Test your integration without training data
   curl -X POST http://localhost:8000/predict \
     -H "Content-Type: application/json" \
     -d '{
       "model_id": "test-model",
       "platform_info": {"software_name": "exp", "software_version": "exp", "hardware_name": "exp"},
       "features": {"exp_runtime": 100.0}
     }'
   ```

3. **Start with synthetic data for development**:
   ```python
   # Generate synthetic training data for testing
   import random

   training_data = [
       {
           "input_size": size,
           "batch_size": batch,
           "runtime_ms": size * 0.05 + batch * 0.2 + random.gauss(0, 5)
       }
       for size in [1024, 2048, 4096]
       for batch in [16, 32, 64]
   ]
   ```

### 4. Invalid Quantile Configuration (400)

**Scenario**: Quantile values are out of range or invalid.

#### Error Response

```json
{
  "error": "Invalid training configuration",
  "message": "Quantile values must be between 0 and 1",
  "invalid_quantiles": [1.5, -0.1]
}
```

#### Causes

- Quantile values outside [0, 1] range
- Empty quantiles list
- Non-numeric quantile values

#### Solutions

1. **Use valid quantile values**:
   ```json
   {
     "training_config": {
       "quantiles": [0.5, 0.9, 0.95, 0.99]  // All between 0 and 1
     }
   }
   ```

2. **Common quantile configurations**:
   ```python
   # P50, P90, P95, P99 (recommended)
   quantiles = [0.5, 0.9, 0.95, 0.99]

   # Full distribution
   quantiles = [0.25, 0.5, 0.75, 0.9, 0.95, 0.99]

   # Median only (simplest)
   quantiles = [0.5]
   ```

3. **Omit configuration for defaults**:
   ```json
   {
     "prediction_type": "quantile",
     // No training_config - uses default [0.5, 0.9, 0.95, 0.99]
   }
   ```

**Scenario**: Training samples don't have `runtime_ms` field.

#### Error Response

```json
{
  "error": "Invalid training data",
  "message": "All training samples must have 'runtime_ms' field",
  "sample_index": 2
}
```

#### Solutions

```json
{
  "features_list": [
    {
      "image_width": 1920,
      "image_height": 1080,
      "runtime_ms": 125.3  // Required!
    }
  ]
}
```

### 6. Service Unavailable (503)

**Scenario**: Service is not ready or experiencing issues.

#### Error Response

```json
{
  "error": "Service unavailable",
  "message": "Service is starting up"
}
```

#### Causes

- Service is still starting up
- Model storage is unavailable
- Internal service error

#### Solutions

1. **Wait and retry**:
   ```bash
   # Check health endpoint
   curl -X GET http://localhost:8000/health

   # Wait a few seconds and retry
   sleep 5
   curl -X GET http://localhost:8000/health
   ```

2. **Check service logs**:
   ```bash
   # Check container/service logs for errors
   docker logs predictor-service
   ```

3. **Verify service is running**:
   ```bash
   # Check if service is accessible
   curl -v http://localhost:8000/health
   ```

## HTTP Status Codes

### Status Code Reference

| Code | Meaning | When It Occurs |
|------|---------|----------------|
| **200 OK** | Success | Request completed successfully |
| **400 Bad Request** | Invalid Input | Malformed JSON, missing fields, invalid values |
| **404 Not Found** | Model Not Found | Model hasn't been trained for this combination |
| **500 Internal Server Error** | Server Error | MLP training failed, internal bug, resource exhaustion |
| **503 Service Unavailable** | Service Unhealthy | Service is starting up or experiencing issues |

### Handling Each Status Code

#### 200 OK
```python
response = requests.post(url, json=data)
if response.status_code == 200:
    result = response.json()
    # Process successful result
```

#### 400 Bad Request
```python
if response.status_code == 400:
    error = response.json()
    print(f"Invalid request: {error['message']}")
    # Fix request parameters and retry
```

#### 404 Not Found
```python
if response.status_code == 404:
    # Model doesn't exist - train it first
    train_model(model_id, platform_info, training_data)
    # Then retry prediction
```

#### 500 Internal Server Error
```python
if response.status_code == 500:
    # Log error and retry with backoff
    logging.error(f"Server error: {response.json()}")
    time.sleep(retry_delay)
    # Retry with exponential backoff
```

#### 503 Service Unavailable
```python
if response.status_code == 503:
    # Service not ready - retry after delay
    time.sleep(5)
    # Check health endpoint before retrying
```

## Error Response Format

All errors follow a consistent format:

```json
{
  "error": "Error category",
  "message": "Detailed human-readable message",
  "field1": "Additional context",
  "field2": "More context"
}
```

### Parsing Errors in Code

```python
import requests

try:
    response = requests.post(url, json=data)
    response.raise_for_status()  # Raises exception for 4xx/5xx
    result = response.json()
except requests.exceptions.HTTPError as e:
    error_data = e.response.json()
    print(f"Error: {error_data.get('error')}")
    print(f"Message: {error_data.get('message')}")
    # Handle specific error types
    if e.response.status_code == 404:
        # Train model
        pass
    elif e.response.status_code == 400:
        # Fix request
        pass
```

## Prevention Best Practices

### 1. Validate Input Before Sending

```python
def validate_prediction_request(request):
    assert "model_id" in request
    assert "platform_info" in request
    assert "prediction_type" in request["prediction_type"] in ["expect_error", "quantile"]
    assert "features" in request
    assert all(isinstance(v, (int, float)) for v in request["features"].values())
```

### 2. Use Consistent Feature Names

```python
# Define feature schema once
FEATURE_SCHEMA = {
    "image_width": int,
    "image_height": int,
    "batch_size": int
}

# Use everywhere
def create_features(width, height, batch):
    return {
        "image_width": width,
        "image_height": height,
        "batch_size": batch
    }
```

### 3. Implement Retry Logic

```python
import time
from requests.adapters import HTTPAdapter
from requests.packages.urllib3.util.retry import Retry

def create_session_with_retries():
    session = requests.Session()
    retry = Retry(
        total=3,
        backoff_factor=1,
        status_forcelist=[500, 502, 503, 504]
    )
    adapter = HTTPAdapter(max_retries=retry)
    session.mount('http://', adapter)
    session.mount('https://', adapter)
    return session

session = create_session_with_retries()
response = session.post(url, json=data)
```

### 4. Log Errors for Analysis

```python
import logging

logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

try:
    response = requests.post(url, json=data)
    response.raise_for_status()
except requests.exceptions.HTTPError as e:
    logger.error(f"Prediction failed: {e.response.status_code}")
    logger.error(f"Request: {data}")
    logger.error(f"Response: {e.response.text}")
    raise
```

### 5. Use Health Checks

```python
def wait_for_service(url, timeout=30):
    start = time.time()
    while time.time() - start < timeout:
        try:
            response = requests.get(f"{url}/health")
            if response.status_code == 200:
                return True
        except requests.exceptions.RequestException:
            pass
        time.sleep(1)
    return False

if wait_for_service("http://localhost:8000"):
    # Service is ready
    make_predictions()
```

### 6. Test with Experiment Mode First

```python
def test_integration():
    # Test with experiment mode before using real models
    response = requests.post(
        "http://localhost:8000/predict",
        json={
            "model_id": "test",
            "platform_info": {
                "software_name": "exp",
                "software_version": "exp",
                "hardware_name": "exp"
            },
            "prediction_type": "expect_error",
            "features": {"exp_runtime": 100.0}
        }
    )
    assert response.status_code == 200
    print("Integration test passed!")
```

## Debugging Checklist

When encountering errors, work through this checklist:

- [ ] Service is running and healthy (`/health` returns 200)
- [ ] Model exists for this configuration (`/list` shows the model)
- [ ] Request JSON is valid (no syntax errors)
- [ ] All required fields are present
- [ ] Feature names match exactly (case-sensitive)
- [ ] Feature values are correct types (numbers vs strings)
- [ ] Platform info matches trained model exactly
- [ ] Prediction type matches trained model
- [ ] Network connectivity is working
- [ ] Service logs don't show errors
