# API Specification

All endpoints are prefixed with `/v1`.

## Instance Management Endpoints

### POST /v1/instance/register

Register a new worker instance. If the instance is already registered, returns success with existing info (duplicate ignored). Triggers background work stealing from existing instances.

**Request Body: `InstanceRegisterRequest`**

| Field | Type | Required | Description |
|---|---|---|---|
| `instance_id` | string | yes | Unique instance identifier |
| `model_id` | string | yes | Model ID this instance serves |
| `endpoint` | string | yes | HTTP endpoint URL |
| `platform_info` | object | yes | Must contain `software_name`, `software_version`, `hardware_name` |

**Response Body: `InstanceRegisterResponse`** `200`

| Field | Type | Description |
|---|---|---|
| `success` | bool | Whether registration succeeded |
| `message` | string | Status message |
| `instance` | Instance | Registered instance object |

**Error Responses**: `400` (missing platform_info keys, duplicate ID)

### POST /v1/instance/remove

Remove an instance. Allows removal even with pending tasks; callbacks from removed instances are still accepted.

**Request Body: `InstanceRemoveRequest`**

| Field | Type | Required | Description |
|---|---|---|---|
| `instance_id` | string | yes | Instance ID to remove |

**Response Body: `InstanceRemoveResponse`** `200`

| Field | Type | Description |
|---|---|---|
| `success` | bool | Whether removal succeeded |
| `message` | string | Status message |
| `instance_id` | string | Removed instance ID |

**Error Responses**: `404` (instance not found)

### POST /v1/instance/drain

Start draining an instance: stop assigning new tasks while existing tasks complete.

**Request Body: `InstanceDrainRequest`**

| Field | Type | Required | Description |
|---|---|---|---|
| `instance_id` | string | yes | Instance ID to drain |

**Response Body: `InstanceDrainResponse`** `200`

| Field | Type | Description |
|---|---|---|
| `success` | bool | Whether drain started |
| `message` | string | Status message |
| `instance_id` | string | Instance ID |
| `status` | InstanceStatus | Current status (should be `draining`) |
| `pending_tasks` | int | Number of pending tasks |
| `running_tasks` | int | Number of running tasks |
| `estimated_completion_time_ms` | float \| null | Estimated time to complete all tasks |

**Error Responses**: `404` (instance not found), `400` (not in ACTIVE state)

### GET /v1/instance/drain/status

Check drain progress for an instance.

**Query Parameters:** `instance_id` (string, required)

**Response Body: `InstanceDrainStatusResponse`** `200`

| Field | Type | Description |
|---|---|---|
| `success` | bool | Request succeeded |
| `instance_id` | string | Instance ID |
| `status` | InstanceStatus | Current status |
| `pending_tasks` | int | Remaining pending tasks |
| `running_tasks` | int | Running tasks |
| `can_remove` | bool | Whether it is safe to remove the instance |
| `drain_initiated_at` | string \| null | ISO timestamp when draining started |

**Error Responses**: `404` (instance not found)

### POST /v1/instance/redeploy/start

Start redeploying an instance. Sets status to REDEPLOYING, extracts pending tasks from the instance, redistributes them to other active instances.

**Request Body: `InstanceRedeployRequest`**

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `instance_id` | string | yes | | Instance ID to redeploy |
| `redeploy_reason` | string | no | `null` | Reason for redeployment |
| `target_model_id` | string | no | `null` | Optional target model ID |

**Response Body: `InstanceRedeployResponse`** `200`

| Field | Type | Description |
|---|---|---|
| `success` | bool | Whether redeployment started |
| `message` | string | Status message with redistribution summary |
| `returned_tasks` | object[] | Tasks returned from the instance |
| `redistributed_tasks` | string[] | Task IDs successfully redistributed |
| `failed_redistributions` | string[] | Task IDs that failed to redistribute |
| `current_task` | object \| null | Currently executing task info |
| `estimated_redeploy_time_ms` | float \| null | Estimated redeployment time |

**Error Responses**: `404` (instance not found), `400` (not in ACTIVE state), `500` (communication failure)

### POST /v1/instance/redeploy/complete

Complete redeployment. Updates instance config and transitions from REDEPLOYING to ACTIVE.

**Request Body: `InstanceRegisterRequest`** (same as register)

| Field | Type | Required | Description |
|---|---|---|---|
| `instance_id` | string | yes | Instance ID |
| `model_id` | string | yes | New model ID |
| `endpoint` | string | yes | New endpoint |
| `platform_info` | object | yes | New platform info |

**Response Body: `InstanceRegisterResponse`** `200`

**Error Responses**: `404` (instance not found), `400` (not in REDEPLOYING state)

### GET /v1/instance/list

List all registered instances with optional filtering.

**Query Parameters:** `model_id` (string, optional)

**Response Body: `InstanceListResponse`** `200`

| Field | Type | Description |
|---|---|---|
| `success` | bool | Request succeeded |
| `count` | int | Number of instances returned |
| `instances` | Instance[] | List of instances |

### GET /v1/instance/info

Get detailed information about a specific instance including queue info and stats.

**Query Parameters:** `instance_id` (string, required)

**Response Body: `InstanceInfoResponse`** `200`

| Field | Type | Description |
|---|---|---|
| `success` | bool | Request succeeded |
| `instance` | Instance | Instance details |
| `queue_info` | InstanceQueueProbabilistic \| InstanceQueueExpectError | Current queue state |
| `stats` | InstanceStats | Task statistics (pending, completed, failed) |

**Error Responses**: `404` (instance not found), `500` (queue info or stats missing)

## Task Management Endpoints

### POST /v1/task/submit

Submit a task for scheduling. Returns immediately after creating the task record. The task is scheduled to a worker queue in the background. If no instances are available, the task remains in PENDING status.

**Request Body: `TaskSubmitRequest`**

| Field | Type | Required | Description |
|---|---|---|---|
| `task_id` | string | yes | Unique task identifier |
| `model_id` | string | yes | Target model ID |
| `task_input` | object | yes | Input data for the model |
| `metadata` | object | yes | Task metadata (used for prediction) |

**Response Body: `TaskSubmitResponse`** `200`

| Field | Type | Description |
|---|---|---|
| `success` | bool | Whether submission succeeded |
| `message` | string | Status message with queue position |
| `task` | TaskInfo | Task info with status and assignment |

**TaskInfo fields:**

| Field | Type | Description |
|---|---|---|
| `task_id` | string | Task identifier |
| `status` | TaskStatus | Current status (initially `pending`) |
| `assigned_instance` | string | Assigned instance ID (empty if not yet scheduled) |
| `submitted_at` | string | ISO timestamp |

**Error Responses**: `400` (duplicate task_id)

### GET /v1/task/list

List tasks with optional filtering and pagination.

**Query Parameters:**

| Parameter | Type | Default | Description |
|---|---|---|---|
| `status` | TaskStatus | *(none)* | Filter by status |
| `model_id` | string | *(none)* | Filter by model ID |
| `instance_id` | string | *(none)* | Filter by assigned instance |
| `limit` | int | `100` | Max results (1-1000) |
| `offset` | int | `0` | Pagination offset |

**Response Body: `TaskListResponse`** `200`

| Field | Type | Description |
|---|---|---|
| `success` | bool | Request succeeded |
| `count` | int | Number of tasks returned |
| `total` | int | Total matching tasks |
| `offset` | int | Current offset |
| `limit` | int | Current limit |
| `tasks` | TaskSummary[] | List of task summaries |

**TaskSummary fields:**

| Field | Type | Description |
|---|---|---|
| `task_id` | string | Task identifier |
| `model_id` | string | Model ID |
| `status` | TaskStatus | Current status |
| `assigned_instance` | string | Assigned instance ID |
| `submitted_at` | string | ISO timestamp |
| `completed_at` | string \| null | Completion timestamp |

### GET /v1/task/info

Get detailed information about a specific task.

**Query Parameters:** `task_id` (string, required)

**Response Body: `TaskDetailResponse`** `200`

| Field | Type | Description |
|---|---|---|
| `success` | bool | Request succeeded |
| `task` | TaskDetailInfo | Full task details |

**TaskDetailInfo fields:**

| Field | Type | Description |
|---|---|---|
| `task_id` | string | Task identifier |
| `model_id` | string | Model ID |
| `status` | TaskStatus | Current status |
| `assigned_instance` | string | Assigned instance |

| `error` | string \| null | Error message |
| `timestamps` | TaskTimestamps | Submission, start, and completion timestamps |
| `execution_time_ms` | float \| null | Actual execution time |

**Error Responses**: `404` (task not found)

### POST /v1/task/clear

Clear all tasks from the scheduler and all registered instances. Resets pending task counters.

**Response Body: `TaskClearResponse`** `200`

| Field | Type | Description |
|---|---|---|
| `success` | bool | Whether clear succeeded |
| `message` | string | Summary message |
| `cleared_count` | int | Number of tasks cleared from scheduler |

### POST /v1/task/resubmit

Resubmit a task during instance migration. Resets task to PENDING, decrements the original instance's pending count, and reschedules to a new instance.

**Request Body: `TaskResubmitRequest`**

| Field | Type | Required | Description |
|---|---|---|---|
| `task_id` | string | yes | Task ID to resubmit |
| `original_instance_id` | string | yes | Original instance ID (for stats update) |

**Response Body: `TaskResubmitResponse`** `200`

| Field | Type | Description |
|---|---|---|
| `success` | bool | Whether resubmission succeeded |
| `message` | string | Status message with queue position |

**Error Responses**: `404` (task or instance not found), `400` (task COMPLETED/FAILED)

### POST /v1/task/update_metadata

Batch update metadata for multiple tasks and re-predict if tasks are in a queue. Implements rollback on prediction failure. COMPLETED and FAILED tasks are silently skipped.

**Request Body: `TaskUpdateMetadataRequest`**

| Field | Type | Required | Description |
|---|---|---|---|
| `updates` | TaskMetadataUpdate[] | yes | List of updates |

**TaskMetadataUpdate fields:**

| Field | Type | Required | Description |
|---|---|---|---|
| `task_id` | string | yes | Task to update |
| `metadata` | object | yes | New metadata (replaces existing) |

**Response Body: `TaskUpdateMetadataResponse`** `200`

| Field | Type | Description |
|---|---|---|
| `success` | bool | True if no failures |
| `message` | string | Summary message |
| `total` | int | Total updates requested |
| `succeeded` | int | Successful updates |
| `failed` | int | Failed updates |
| `skipped` | int | Skipped (COMPLETED/FAILED tasks) |
| `results` | TaskUpdateMetadataResult[] | Per-task results |

### POST /v1/task/repredict

Re-predict all pending/running tasks without changing metadata. Use after predictor model retraining.

**Response Body: `TaskRepredictResponse`** `200`

| Field | Type | Description |
|---|---|---|
| `success` | bool | True if no failures |
| `message` | string | Summary message |
| `total_tasks` | int | Total tasks in registry |
| `eligible_tasks` | int | Tasks eligible for re-prediction |
| `repredicted` | int | Successfully re-predicted |
| `failed` | int | Failed re-predictions |
| `skipped` | int | Skipped tasks |

**TaskScheduleInfo fields:**

## WebSocket Protocol

### Connection

```
ws://host:port/v1/task/get_result
```

### Message Types

| Type | Direction | Purpose |
|---|---|---|
| `subscribe` | Client → Server | Subscribe to task result notifications |
| `unsubscribe` | Client → Server | Unsubscribe from task notifications |
| `ack` | Server → Client | Acknowledgment of subscribe/unsubscribe |
| `result` | Server → Client | Task result notification |
| `error` | Server → Client | Error message |
| `ping` | Server → Client | Keepalive ping (every 10s) |
| `pong` | Client → Server | Keepalive pong response |

### Subscribe Message (Client → Server)

```json
{
  "type": "subscribe",
  "task_ids": ["task-001", "task-002"]
}
```

### Unsubscribe Message (Client → Server)

```json
{
  "type": "unsubscribe",
  "task_ids": ["task-001"]
}
```

### Ack Message (Server → Client)

```json
{
  "type": "ack",
  "message": "Subscribed to 2 tasks",
  "subscribed_tasks": ["task-001", "task-002"]
}
```

### Result Message (Server → Client)

```json
{
  "type": "result",
  "task_id": "task-001",
  "status": "completed",
  "result": {"output": "..."},
  "error": null,
  "timestamps": {
    "submitted_at": "2026-01-30T12:00:00",
    "started_at": "2026-01-30T12:00:01",
    "completed_at": "2026-01-30T12:00:05"
  },
  "execution_time_ms": 4000.0
}
```

### Error Message (Server → Client)

```json
{
  "type": "error",
  "error": "Invalid JSON format",
  "task_id": null
}
```

### Ping/Pong Messages

```json
{"type": "ping", "timestamp": 1706612400.0}
{"type": "pong", "timestamp": 1706612400.0}
```

**Behavior notes:**
- If a subscribed task is already COMPLETED or FAILED, the result is sent immediately on subscribe.
- The server sends application-level ping messages every 10 seconds. The client should respond with pong.
- Both protocol-level and application-level ping/pong are supported.

## Transparent Proxy

The catch-all route `/{path}` forwards any unmatched HTTP request through the scheduling pipeline to a backend instance.

**Flow:**
1. Parse request body and headers
2. Select instance via scheduling algorithm
3. Enqueue task to the selected instance's `WorkerQueueThread`
4. Await result (with configurable timeout, default 300s)
5. Return the backend's response transparently (status code, body, headers)

**Supported methods:** GET, POST, PUT, DELETE, PATCH, OPTIONS, HEAD

**Error responses:**
- `503`: No instances available or instance not ready
- `504`: Request timed out
- `502`: Backend request failed (connection error)

## Shared Data Models

### Instance

| Field | Type | Description |
|---|---|---|
| `instance_id` | string | Unique identifier |
| `model_id` | string | Model this instance serves |
| `endpoint` | string | HTTP endpoint URL |
| `platform_info` | object | `{software_name, software_version, hardware_name}` |
| `status` | InstanceStatus | Current lifecycle status |
| `drain_initiated_at` | string \| null | ISO timestamp when drain started |

### InstanceStatus (enum)

| Value | Description |
|---|---|
| `initializing` | Registered, work stealing in progress, no new tasks yet |
| `active` | Normal operation, accepts new tasks |
| `draining` | No new tasks, waiting for existing tasks to complete |
| `removing` | All tasks complete, safe to remove |
| `redeploying` | Being redeployed, pending tasks returned |

### TaskStatus (enum)

| Value | Description |
|---|---|
| `pending` | Waiting for execution |
| `running` | Currently executing on an instance |
| `completed` | Successfully completed |
| `failed` | Execution failed |

### StrategyType (enum)

| Value | Description |
|---|---|
| `adaptive_bootstrap` | Cold/warm auto-detection (default) |
| `probabilistic` | Monte Carlo quantile-based selection |
| `min_time` | Greedy minimum expected time |
| `po2` | Power of Two Choices |
| `round_robin` | Cyclic distribution |
| `random` | Uniform random selection |
| `serverless` | Min-time variant for serverless |

### InstanceQueueProbabilistic

| Field | Type | Description |
|---|---|---|
| `instance_id` | string | Instance identifier |
| `quantiles` | float[] | Quantile levels (e.g., [0.5, 0.9, 0.95, 0.99]) |
| `values` | float[] | Corresponding time values in ms |

### InstanceQueueExpectError

| Field | Type | Description |
|---|---|---|
| `instance_id` | string | Instance identifier |
| `expected_time_ms` | float | Expected queue completion time |
| `error_margin_ms` | float | Error margin (standard deviation) |

### TaskTimestamps

| Field | Type | Description |
|---|---|---|
| `submitted_at` | string | ISO timestamp when task was submitted |
| `started_at` | string \| null | ISO timestamp when execution started |
| `completed_at` | string \| null | ISO timestamp when execution completed |

## Error Handling

| Status Code | Meaning |
|---|---|
| `200` | Success |
| `400` | Invalid input (validation error, duplicate ID, invalid state) |
| `404` | Resource not found (instance, task) |
| `422` | Request body validation error (Pydantic) |
| `500` | Internal error (queue info missing, communication failure) |
| `502` | Bad gateway (proxy: backend request failed) |
| `503` | Service unavailable (proxy: no instances, instance not ready) |
| `504` | Gateway timeout (proxy: request timed out) |

Error response format:

```json
{"success": false, "error": "Error description"}
```
