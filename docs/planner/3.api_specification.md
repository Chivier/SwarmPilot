# API Specification

All endpoints are prefixed with `/v1`.

## Endpoint Quick Reference

### Core

| Method | Path | Purpose |
|---|---|---|
| GET | `/v1/health` | Health check |
| GET | `/v1/info` | Service info and capabilities |
| POST | `/v1/plan` | Compute optimal deployment (no execution) |

### Instance Registration

| Method | Path | Purpose |
|---|---|---|
| POST | `/v1/instance/register` | Register an available instance |

### Scheduler Registry

| Method | Path | Purpose |
|---|---|---|
| POST | `/v1/scheduler/register` | Register a scheduler for a model |
| POST | `/v1/scheduler/deregister` | Deregister a scheduler |
| GET | `/v1/scheduler/list` | List all registered schedulers |
| GET | `/v1/scheduler/{model_id}` | Get scheduler for a specific model |

### PyLet Deployment

| Method | Path | Purpose |
|---|---|---|
| GET | `/v1/status` | PyLet service status |
| POST | `/v1/deploy_manually` | Deploy to manually specified target state |
| POST | `/v1/deploy` | Run optimizer then deploy via PyLet |
| POST | `/v1/scale` | Scale a specific model |
| POST | `/v1/migrate` | Migrate an instance (cancel-and-resubmit) |
| POST | `/v1/optimize` | Run optimizer and deploy (simplified input) |
| POST | `/v1/terminate-all` | Terminate all PyLet instances |

### Scheduler Compatibility Stubs

| Method | Path | Purpose |
|---|---|---|
| POST | `/v1/instance/drain` | Stub: acknowledge drain (always succeeds) |
| GET | `/v1/instance/drain/status` | Stub: drain status (always `can_remove=true`) |
| POST | `/v1/instance/remove` | Stub: acknowledge removal (always succeeds) |
| POST | `/v1/task/resubmit` | Stub: acknowledge task resubmit (always succeeds) |

### Timeline

| Method | Path | Purpose |
|---|---|---|
| GET | `/v1/timeline` | Get instance count timeline |
| POST | `/v1/timeline/clear` | Clear the timeline |

## Core Endpoints

### GET /v1/health

Health check for monitoring and load balancing.

**Response** `200`

```json
{"status": "healthy", "timestamp": "2026-01-30T12:00:00+00:00"}
```

### GET /v1/info

Service metadata including version, supported algorithms, PyLet status, and scheduler registry.

**Response** `200`

```json
{
  "service": "planner",
  "version": "0.1.0",
  "algorithms": ["simulated_annealing", "integer_programming"],
  "objective_methods": ["relative_error", "ratio_difference", "weighted_squared"],
  "description": "Model deployment optimization service",
  "available_instances": [],
  "pylet": {
    "enabled": true,
    "initialized": true,
    "active_instances": 3,
    "current_state": {"model-a": 2, "model-b": 1}
  },
  "scheduler_registry": {
    "total": 2,
    "models": ["model-a", "model-b"]
  }
}
```

### POST /v1/plan

Compute optimal deployment without execution.

**Request Body: `PlannerInput`**

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `M` | int | yes | | Number of instances (>0) |
| `N` | int | yes | | Number of model types (>0) |
| `B` | float[][] | yes | | Batch capacity matrix [M x N] |
| `initial` | int[] \| null | no | null | Initial deployment [M], -1 = no model |
| `a` | float | yes | | Change constraint (0 < a <= 1) |
| `target` | float[] | yes | | Target request distribution [N] |
| `algorithm` | string | no | `"simulated_annealing"` | `"simulated_annealing"` or `"integer_programming"` |
| `objective_method` | string | no | `"relative_error"` | `"relative_error"`, `"ratio_difference"`, or `"weighted_squared"` |
| `verbose` | bool | no | `true` | Enable detailed logging |
| `initial_temp` | float | no | `100.0` | SA: starting temperature (>0) |
| `final_temp` | float | no | `0.01` | SA: ending temperature (>0, < initial_temp) |
| `cooling_rate` | float | no | `0.95` | SA: temperature decay (0 < rate < 1) |
| `max_iterations` | int | no | `5000` | SA: maximum iterations (>0) |
| `iterations_per_temp` | int | no | `100` | SA: iterations per temperature level (>0) |
| `solver_name` | string | no | `"PULP_CBC_CMD"` | IP: solver backend name |
| `time_limit` | int | no | `300` | IP: solver timeout in seconds (>0) |

**Response Body: `PlannerOutput`** `200`

| Field | Type | Description |
|---|---|---|
| `deployment` | int[] | Optimized model assignment [M] |
| `score` | float | Objective value (lower is better) |
| `stats` | object | Algorithm-specific statistics |
| `service_capacity` | float[] | Total throughput per model [N] |
| `changes_count` | int | Number of instances changed from initial |

**Error Responses**: `400` (invalid input), `500` (optimization failure)

## Instance Registration

### POST /v1/instance/register

Register an available instance with the planner's store. Used for migration-based redeployment.

**Request Body: `InstanceRegisterRequest`**

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `instance_id` | string | yes | | Unique instance identifier |
| `model_id` | string | yes | | Model ID supported by this instance |
| `endpoint` | string | yes | | Instance HTTP endpoint URL |
| `platform_info` | object | no | `{}` | Platform metadata (software_name, software_version, hardware_name) |

**Response Body: `InstanceRegisterResponse`** `200`

| Field | Type | Description |
|---|---|---|
| `success` | bool | Whether registration succeeded |
| `message` | string | Status message |

## Scheduler Registry Endpoints

### POST /v1/scheduler/register

Register a scheduler for a specific model. Replaces any previous registration for the same model.

**Request Body: `SchedulerRegisterRequest`**

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `model_id` | string | yes | | Model identifier |
| `scheduler_url` | string | yes | | Base URL of the scheduler |
| `metadata` | object | no | `{}` | Optional metadata |

**Response Body: `SchedulerRegisterResponse`** `200`

| Field | Type | Description |
|---|---|---|
| `success` | bool | Whether registration succeeded |
| `message` | string | Status message |
| `replaced_previous` | bool | Whether a previous registration was replaced |

### POST /v1/scheduler/deregister

Deregister a scheduler for a model.

**Request Body: `SchedulerDeregisterRequest`**

| Field | Type | Required | Description |
|---|---|---|---|
| `model_id` | string | yes | Model identifier to deregister |

**Response Body: `SchedulerDeregisterResponse`** `200`

| Field | Type | Description |
|---|---|---|
| `success` | bool | Whether deregistration succeeded |
| `message` | string | Status message |

### GET /v1/scheduler/list

List all registered schedulers.

**Response Body: `SchedulerListResponse`** `200`

| Field | Type | Description |
|---|---|---|
| `schedulers` | SchedulerInfo[] | List of registered schedulers |
| `total` | int | Total count |

**`SchedulerInfo` fields:**

| Field | Type | Description |
|---|---|---|
| `model_id` | string | Model identifier |
| `scheduler_url` | string | Scheduler base URL |
| `registered_at` | string | ISO timestamp |
| `is_healthy` | bool | Health status |
| `metadata` | object | Optional metadata |

### GET /v1/scheduler/{model_id}

Get the scheduler registered for a specific model.

**Path Parameters:** `model_id` (string)

**Response** `200`: `SchedulerInfo` object (see above)

**Error Response**: `404` if no scheduler registered for the model.

## PyLet Endpoints

All PyLet endpoints return `503` if PyLet is not enabled or not initialized.

### GET /v1/status

Get PyLet service status and active instances.

**Response Body: `PyLetStatusOutput`** `200`

| Field | Type | Description |
|---|---|---|
| `enabled` | bool | Whether PyLet is enabled |
| `initialized` | bool | Whether PyLet is initialized |
| `current_state` | object | Model counts `{model_id: count}` |
| `total_instances` | int | Total managed instances |
| `active_instances` | PyLetInstanceStatus[] | All active instances |

### POST /v1/deploy_manually

Deploy instances to a manually specified target state. Reconciles current state to target by adding/removing instances.

**Request Body: `PyLetDeploymentInput`**

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `target_state` | object | yes | | Target model counts `{model_id: count}` |
| `wait_for_ready` | bool | no | `true` | Wait for instances to become ready |
| `register_with_scheduler` | bool | no | `true` | Register new instances with scheduler |

**Response Body: `PyLetDeploymentOutput`** `200`

| Field | Type | Description |
|---|---|---|
| `success` | bool | Overall success flag |
| `added_count` | int | Instances added |
| `removed_count` | int | Instances removed |
| `active_instances` | PyLetInstanceStatus[] | Currently active instances |
| `failed_adds` | [string, string][] | Failed add operations (model_id, error) |
| `failed_removes` | [string, string][] | Failed remove operations (pylet_id, error) |
| `error` | string \| null | Error message if failed |

### POST /v1/deploy

Run the optimization algorithm then deploy the result via PyLet. Accepts all `PlannerInput` fields plus deployment-specific fields.

**Request Body: `PyLetDeployWithPlanInput`** (extends `PlannerInput`)

All fields from `PlannerInput` (see `/v1/plan`) plus:

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `model_ids` | string[] | yes | | Model IDs in order (maps indices 0..N-1 to model names) |
| `wait_for_ready` | bool | no | `true` | Wait for instances after deployment |

**Response Body: `PyLetOptimizeOutput`** `200`

| Field | Type | Description |
|---|---|---|
| `deployment` | int[] | Optimized assignment [M] |
| `score` | float | Objective value |
| `service_capacity` | float[] | Capacity per model [N] |
| `changes_count` | int | Changes from initial |
| `stats` | object | Algorithm statistics |
| `deployment_success` | bool | Whether PyLet deployment succeeded |
| `added_count` | int | Instances added |
| `removed_count` | int | Instances removed |
| `active_instances` | PyLetInstanceStatus[] | Active instances after deployment |
| `error` | string \| null | Error if deployment failed |

**Error Responses**: `400` (invalid input, unknown algorithm, unregistered model_ids), `500` (optimization or deployment failure), `503` (PyLet unavailable)

### POST /v1/scale

Scale a specific model to a target instance count.

**Request Body: `PyLetScaleInput`**

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `model_id` | string | yes | | Model to scale |
| `target_count` | int | yes | | Target instance count (>=0) |
| `wait_for_ready` | bool | no | `true` | Wait for new instances |

**Response Body: `PyLetScaleOutput`** `200`

| Field | Type | Description |
|---|---|---|
| `success` | bool | Overall success flag |
| `model_id` | string | Model that was scaled |
| `previous_count` | int | Previous instance count |
| `current_count` | int | Current instance count |
| `added` | int | Instances added |
| `removed` | int | Instances removed |
| `active_instances` | PyLetInstanceStatus[] | Active instances for this model |
| `error` | string \| null | Error message if failed |

### POST /v1/migrate

Migrate an instance via cancel-and-resubmit. Terminates the old instance and deploys a new one.

**Request Body: `PyLetMigrateInput`**

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `pylet_id` | string | yes | | PyLet ID of instance to migrate |
| `target_model_id` | string \| null | no | `null` | New model ID (null = keep same) |

**Response Body: `PyLetMigrateOutput`** `200`

| Field | Type | Description |
|---|---|---|
| `success` | bool | Overall success flag |
| `old_pylet_id` | string | Original PyLet ID |
| `new_pylet_id` | string \| null | New PyLet ID |
| `model_id` | string | Model ID |
| `endpoint` | string \| null | New endpoint |
| `error` | string \| null | Error message if failed |

### POST /v1/optimize

Run optimizer and deploy via PyLet using simplified input (no `initial` array, no SA/IP tuning parameters).

**Request Body: `PyLetOptimizeInput`**

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `target` | float[] | yes | | Target distribution |
| `model_ids` | string[] | yes | | Model IDs in order |
| `B` | float[][] | yes | | Capacity matrix |
| `a` | float | no | `0.3` | Change penalty factor (0 to 1) |
| `objective_method` | string | no | `"ratio_difference"` | Objective function |
| `algorithm` | string | no | `"simulated_annealing"` | Optimization algorithm |
| `wait_for_ready` | bool | no | `true` | Wait for instances |

**Response Body: `PyLetOptimizeOutput`** `200` — same as `/v1/deploy` response.

### POST /v1/terminate-all

Terminate all PyLet-managed instances.

**Query Parameters:**

| Parameter | Type | Default | Description |
|---|---|---|---|
| `wait_for_drain` | bool | `false` | Wait for scheduler drain before termination |

**Response** `200`

```json
{
  "success": true,
  "total": 5,
  "succeeded": 5,
  "failed": 0,
  "details": {"pylet-id-1": true, "pylet-id-2": true}
}
```

## Scheduler Compatibility Stubs

These endpoints exist so that instances registered with the Planner (instead of a Scheduler) can complete their deregistration flow without errors. They always return success since the Planner does not manage task queues.

### POST /v1/instance/drain

**Request Body: `InstanceDrainRequest`**

| Field | Type | Required | Description |
|---|---|---|---|
| `instance_id` | string | yes | Instance ID to drain |

**Response Body: `InstanceDrainResponse`** `200`

| Field | Type | Description |
|---|---|---|
| `success` | bool | Always `true` |
| `message` | string | Acknowledgment message |
| `instance_id` | string | Instance ID |
| `status` | string | Always `"draining"` |
| `pending_tasks` | int | Always `0` |
| `running_tasks` | int | Always `0` |
| `estimated_completion_time_ms` | float \| null | Always `0.0` |

### GET /v1/instance/drain/status

**Query Parameters:** `instance_id` (string)

**Response Body: `InstanceDrainStatusResponse`** `200`

| Field | Type | Description |
|---|---|---|
| `success` | bool | Always `true` |
| `instance_id` | string | Instance ID |
| `status` | string | Always `"removing"` |
| `pending_tasks` | int | Always `0` |
| `running_tasks` | int | Always `0` |
| `can_remove` | bool | Always `true` |
| `drain_initiated_at` | string \| null | Always `null` |

### POST /v1/instance/remove

**Request Body: `InstanceRemoveRequest`**

| Field | Type | Required | Description |
|---|---|---|---|
| `instance_id` | string | yes | Instance ID to remove |

**Response Body: `InstanceRemoveResponse`** `200`

| Field | Type | Description |
|---|---|---|
| `success` | bool | Always `true` |
| `message` | string | Acknowledgment message |
| `instance_id` | string | Instance ID |

### POST /v1/task/resubmit

**Request Body: `TaskResubmitRequest`**

| Field | Type | Required | Description |
|---|---|---|---|
| `task_id` | string | yes | Task ID to resubmit |
| `original_instance_id` | string | yes | Original instance ID |

**Response Body: `TaskResubmitResponse`** `200`

| Field | Type | Description |
|---|---|---|
| `success` | bool | Always `true` |
| `message` | string | Acknowledgment message |

## Timeline Endpoints

### GET /v1/timeline

Get the instance count timeline. Returns all recorded migration events with instance counts per model.

**Response** `200`

```json
{
  "success": true,
  "entry_count": 12,
  "entries": [...]
}
```

### POST /v1/timeline/clear

Clear the timeline. Call at the start of a new experiment for clean data.

**Response** `200`

```json
{"success": true, "message": "Timeline cleared successfully"}
```

## Shared Data Models

### PyLetInstanceStatus

Used across all PyLet endpoint responses.

| Field | Type | Description |
|---|---|---|
| `pylet_id` | string | PyLet instance ID |
| `instance_id` | string | Scheduler instance ID |
| `model_id` | string | Model identifier |
| `endpoint` | string \| null | Instance endpoint |
| `status` | string | Instance status |
| `error` | string \| null | Error message if failed |

### InstanceStatus (enum)

| Value | Description |
|---|---|
| `initializing` | Instance is starting up |
| `active` | Instance is serving traffic |
| `draining` | Instance is draining tasks |
| `removing` | Instance is being removed |
| `redeploying` | Instance is being redeployed |

## Error Handling

| Status Code | Meaning |
|---|---|
| `200` | Success |
| `400` | Invalid input (validation error, unknown algorithm) |
| `404` | Resource not found (e.g., no scheduler for model) |
| `422` | Request body validation error (Pydantic) |
| `500` | Internal error (optimization failure, deployment error) |
| `503` | PyLet not enabled or not initialized |

Error response format:

```json
{"detail": "Error description string"}
```

Global exception handler format:

```json
{"success": false, "error": "Internal server error: ..."}
```
