# Quick Start

## Prerequisites

- Python >= 3.11
- [uv](https://docs.astral.sh/uv/) package manager
- (Optional) A running PyLet cluster for deployment mode
- (Optional) A running Scheduler service for instance registration

## Installation

```bash
cd planner
uv sync
```

## Configuration

All settings are controlled via environment variables. Default values work for plan-only mode.

### Basic

| Variable | Default | Description |
|---|---|---|
| `PLANNER_HOST` | `0.0.0.0` | Bind address |
| `PLANNER_PORT` | `8000` | Bind port |
| `SCHEDULER_URL` | *(none)* | Default scheduler URL for instance registration |
| `INSTANCE_TIMEOUT` | `30` | HTTP timeout for instance operations (seconds) |
| `INSTANCE_MAX_RETRIES` | `3` | Max retries for instance operations |
| `INSTANCE_RETRY_DELAY` | `1.0` | Delay between retries (seconds) |

### Auto-Optimize (reserved for future use)

| Variable | Default | Description |
|---|---|---|
| `AUTO_OPTIMIZE_ENABLED` | `false` | Enable automatic periodic optimization |
| `AUTO_OPTIMIZE_INTERVAL` | `60.0` | Interval between auto-optimization runs (seconds) |

### PyLet Core

| Variable | Default | Description |
|---|---|---|
| `PYLET_ENABLED` | `false` | Enable PyLet cluster management |
| `PYLET_HEAD_URL` | *(none)* | PyLet head node URL (required when `PYLET_ENABLED=true`) |
| `PYLET_BACKEND` | `vllm` | Default model backend (`vllm` or `sglang`) |
| `PYLET_GPU_COUNT` | `1` | Default GPUs per instance |

### PyLet Advanced

| Variable | Default | Description |
|---|---|---|
| `PYLET_CPU_COUNT` | `1` | Default CPU cores per instance |
| `PYLET_DEPLOY_TIMEOUT` | `300.0` | Timeout waiting for instance readiness (seconds) |
| `PYLET_DRAIN_TIMEOUT` | `30.0` | Timeout waiting for scheduler drain (seconds) |
| `PYLET_CUSTOM_COMMAND` | *(none)* | Custom launch command template (overrides backend). Use `$PORT` for auto-allocated port |
| `PYLET_REUSE_CLUSTER` | `false` | Reuse existing PyLet connection instead of re-initializing |

## Starting the Service

```bash
uv run splanner start
```

Or run directly with Uvicorn:

```bash
uv run uvicorn src.api:app --host 0.0.0.0 --port 8000
```

With PyLet enabled:

```bash
PYLET_ENABLED=true \
PYLET_HEAD_URL=http://pylet-head:8000 \
SCHEDULER_URL=http://scheduler:8001 \
uv run splanner start
```

## First Optimization

Compute an optimal deployment for 4 instances across 3 models:

```bash
curl -X POST http://localhost:8000/v1/plan \
  -H "Content-Type: application/json" \
  -d '{
    "M": 4,
    "N": 3,
    "B": [[10,5,0],[8,6,4],[0,10,8],[6,0,12]],
    "initial": [0,1,2,2],
    "a": 0.5,
    "target": [20,30,25],
    "algorithm": "simulated_annealing",
    "objective_method": "relative_error"
  }'
```

Response:

```json
{
  "deployment": [0, 1, 1, 2],
  "score": 0.023,
  "stats": {
    "algorithm": "simulated_annealing",
    "iterations": 5000,
    "acceptance_rate": 0.42,
    "final_score": 0.023
  },
  "service_capacity": [10.0, 16.0, 12.0],
  "changes_count": 1
}
```

- **deployment**: Model index assigned to each instance.
- **score**: Objective value (lower is better).
- **service_capacity**: Total throughput per model after deployment.
- **changes_count**: Number of instances that changed from the initial state.

## PyLet Quick Start

With PyLet enabled, deploy the optimization result to real instances:

```bash
curl -X POST http://localhost:8000/v1/deploy \
  -H "Content-Type: application/json" \
  -d '{
    "M": 4,
    "N": 3,
    "B": [[10,5,0],[8,6,4],[0,10,8],[6,0,12]],
    "a": 0.5,
    "target": [20,30,25],
    "model_ids": ["model-a", "model-b", "model-c"],
    "algorithm": "simulated_annealing",
    "wait_for_ready": true
  }'
```

Check cluster status:

```bash
curl http://localhost:8000/v1/status
```
