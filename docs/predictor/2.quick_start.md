# Quick Start

## Prerequisites

- Python >= 3.11
- [uv](https://docs.astral.sh/uv/) package manager
- (Optional) NVIDIA GPU with CUDA for faster training

## Installation

```bash
cd predictor
uv sync
```

## Configuration

All settings are controlled via environment variables with the `PREDICTOR_` prefix. Default values work out of the box.

### Server

| Variable | Default | Description |
|---|---|---|
| `PREDICTOR_HOST` | `0.0.0.0` | Bind address |
| `PREDICTOR_PORT` | `8000` | Bind port |
| `PREDICTOR_RELOAD` | `false` | Enable auto-reload for development |
| `PREDICTOR_WORKERS` | `1` | Number of worker processes (production) |

### Storage

| Variable | Default | Description |
|---|---|---|
| `PREDICTOR_STORAGE_DIR` | `models` | Directory to store trained model files |

### Logging

| Variable | Default | Description |
|---|---|---|
| `PREDICTOR_LOG_LEVEL` | `info` | Logging level (debug, info, warning, error, critical) |
| `PREDICTOR_LOG_DIR` | `logs` | Directory for log files |

### Application

| Variable | Default | Description |
|---|---|---|
| `PREDICTOR_APP_NAME` | `Runtime Predictor Service` | Application name |
| `PREDICTOR_APP_VERSION` | `0.1.0` | Application version |

### Configuration File

Settings can also be provided via a `predictor.toml` file. The service searches for this file in the current and parent directories. See `predictor.toml.example` for the format.

Configuration priority: CLI arguments > Environment variables > TOML file > Defaults.

## Starting the Service

```bash
uv run spredictor start
```

Or run directly with Uvicorn:

```bash
uv run uvicorn src.api.app:app --host 0.0.0.0 --port 8000
```

With custom storage directory:

```bash
PREDICTOR_STORAGE_DIR=/data/models uv run spredictor start --port 9000
```

## First Training

Train a model with sample data (minimum 10 samples required):

```bash
curl -X POST http://localhost:8000/train \
  -H "Content-Type: application/json" \
  -d '{
    "model_id": "llama-7b",
    "platform_info": {
      "software_name": "vllm",
      "software_version": "0.4.0",
      "hardware_name": "NVIDIA A100"
    },
    "prediction_type": "expect_error",
    "features_list": [
      {"batch_size": 1, "seq_len": 128, "runtime_ms": 45.2},
      {"batch_size": 1, "seq_len": 256, "runtime_ms": 82.1},
      {"batch_size": 1, "seq_len": 512, "runtime_ms": 155.3},
      {"batch_size": 2, "seq_len": 128, "runtime_ms": 58.7},
      {"batch_size": 2, "seq_len": 256, "runtime_ms": 105.4},
      {"batch_size": 2, "seq_len": 512, "runtime_ms": 198.6},
      {"batch_size": 4, "seq_len": 128, "runtime_ms": 75.0},
      {"batch_size": 4, "seq_len": 256, "runtime_ms": 140.2},
      {"batch_size": 4, "seq_len": 512, "runtime_ms": 270.8},
      {"batch_size": 8, "seq_len": 128, "runtime_ms": 110.5}
    ]
  }'
```

Response:

```json
{
  "status": "success",
  "message": "Model trained successfully with 10 samples",
  "model_key": "llama-7b__vllm-0.4.0__NVIDIA A100",
  "samples_trained": 10
}
```

- **model_key**: Storage identifier combining model_id, software, and hardware.
- **samples_trained**: Number of training samples used.

## First Prediction

```bash
curl -X POST http://localhost:8000/predict \
  -H "Content-Type: application/json" \
  -d '{
    "model_id": "llama-7b",
    "platform_info": {
      "software_name": "vllm",
      "software_version": "0.4.0",
      "hardware_name": "NVIDIA A100"
    },
    "prediction_type": "expect_error",
    "features": {"batch_size": 4, "seq_len": 256}
  }'
```

Response:

```json
{
  "model_id": "llama-7b",
  "platform_info": {
    "software_name": "vllm",
    "software_version": "0.4.0",
    "hardware_name": "NVIDIA A100"
  },
  "prediction_type": "expect_error",
  "result": {
    "expected_runtime_ms": 138.5,
    "error_margin_ms": 12.3
  }
}
```

## Experiment Mode

Test predictions without training by using the `exp_runtime` feature:

```bash
curl -X POST http://localhost:8000/predict \
  -H "Content-Type: application/json" \
  -d '{
    "model_id": "any-model",
    "platform_info": {
      "software_name": "exp",
      "software_version": "exp",
      "hardware_name": "exp"
    },
    "prediction_type": "expect_error",
    "features": {"exp_runtime": 100.0, "batch_size": 4}
  }'
```

## CLI Commands

```bash
uv run spredictor start      # Start the service
uv run spredictor health     # Check service health
uv run spredictor version    # Show version
uv run spredictor list       # List stored models
uv run spredictor config show   # Show current configuration
uv run spredictor config init   # Generate example config file
```

```bash
uv run pytest tests/ -k "predictor"  # Filter by keyword
uv run pytest --cov=src              # With coverage report
```
