# API Specification

All endpoints are at the root path (no prefix).

## Endpoint Quick Reference

### Core

| Method | Path | Purpose |
|---|---|---|
| GET | `/health` | Health check |
| GET | `/list` | List all trained models |

### Training

| Method | Path | Purpose |
|---|---|---|
| POST | `/train` | Train or retrain a model |

### Prediction

| Method | Path | Purpose |
|---|---|---|
| POST | `/predict` | Make a runtime prediction |
| WS | `/ws/predict` | WebSocket for streaming predictions |

### Cache Management

| Method | Path | Purpose |
|---|---|---|
| GET | `/cache/stats` | Get cache hit/miss statistics |
| POST | `/cache/clear` | Clear the in-memory model cache |

---

## Core Endpoints

### GET /health

Health check for monitoring and load balancing. Returns `503` if storage is not accessible.

**Response** `200`

```json
{"status": "healthy"}
```

**Response** `503`

```json
{"status": "unhealthy", "reason": "Storage directory not accessible: models"}
```

### GET /list

List all trained models with their metadata.

**Response Body: `ModelListResponse`** `200`

| Field | Type | Description |
|---|---|---|
| `models` | ModelMetadata[] | List of all trained models |

**`ModelMetadata` fields:**

| Field | Type | Description |
|---|---|---|
| `model_id` | string | Model identifier |
| `platform_info` | PlatformInfo | Platform information |
| `prediction_type` | string | `"expect_error"` or `"quantile"` |
| `samples_count` | int | Number of training samples |
| `last_trained` | string | ISO 8601 timestamp |

## Training Endpoint

### POST /train

Train or update a model. Requires at least 10 training samples. Each sample must include a `runtime_ms` field. Retraining an existing model overwrites the previous weights.

**Request Body: `TrainingRequest`**

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `model_id` | string | yes | | Unique model identifier |
| `platform_info` | PlatformInfo | yes | | Execution environment |
| `prediction_type` | string | yes | | `"expect_error"` or `"quantile"` |
| `features_list` | object[] | yes | | Training samples (each must have `runtime_ms`) |
| `training_config` | object | no | `null` | Training hyperparameters (see below) |
| `enable_preprocessors` | string[] | no | `null` | Preprocessors to apply |
| `preprocessor_mappings` | object | no | `null` | Feature-to-preprocessor mappings |

**`training_config` fields (common):**

| Field | Type | Default | Description |
|---|---|---|---|
| `epochs` | int | `500` | Number of training epochs |
| `learning_rate` | float | `0.01` | Adam optimizer learning rate |
| `hidden_layers` | int[] | `[64, 32]` | MLP hidden layer sizes |

**`training_config` fields (quantile-specific):**

| Field | Type | Default | Description |
|---|---|---|---|
| `quantiles` | float[] | `[0.5, 0.9, 0.95, 0.99]` | Quantile levels to predict |
| `monotonicity_penalty` | float | `0.0` | Weight for monotonicity constraint |
| `delta_scale` | float | `1.0` | Scale factor for delta outputs |
| `data_augmentation` | object | see below | Data augmentation settings |
| `residual_calibration` | object | see below | Residual calibration settings |
| `log_transform` | object | see below | Log transform settings |

**`data_augmentation` fields:**

| Field | Type | Default | Description |
|---|---|---|---|
| `enabled` | bool | `true` | Enable data augmentation |
| `cv` | float | auto | Coefficient of variation (auto-calculated if omitted) |
| `samples_per_point` | int | `5` | Augmented samples per original sample |
| `distribution` | string | `"lognormal"` | `"lognormal"` or `"normal"` |

**`residual_calibration` fields:**

| Field | Type | Default | Description |
|---|---|---|---|
| `enabled` | bool | `false` | Enable residual calibration |
| `min_sigma` | float | `0.1` | Minimum residual sigma |

**`log_transform` fields:**

| Field | Type | Default | Description |
|---|---|---|---|
| `enabled` | bool | `false` | Apply log transform to runtime_ms |

**Response Body: `TrainingResponse`** `200`

| Field | Type | Description |
|---|---|---|
| `status` | string | `"success"` or `"error"` |
| `message` | string | Detailed result message |
| `model_key` | string | Storage key for the trained model |
| `samples_trained` | int | Number of samples used |

**Error Responses**: `400` (insufficient samples, invalid prediction type, preprocessor error, training validation error), `500` (training failure, save failure)

## Prediction Endpoints

### POST /predict

Make a runtime prediction using a trained model. Automatically uses experiment mode when `exp_runtime` is in features or all `platform_info` fields are `"exp"`.

**Request Body: `PredictionRequest`**

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `model_id` | string | yes | | Model identifier |
| `platform_info` | PlatformInfo | yes | | Execution environment |
| `prediction_type` | string | yes | | `"expect_error"` or `"quantile"` |
| `features` | object | yes | | Feature values for prediction |
| `quantiles` | float[] | no | `null` | Custom quantiles (experiment mode only) |
| `enable_preprocessors` | string[] | no | `null` | Preprocessors to apply |
| `preprocessor_mappings` | object | no | `null` | Feature-to-preprocessor mappings |

**Response Body: `PredictionResponse`** `200`

| Field | Type | Description |
|---|---|---|
| `model_id` | string | Model identifier |
| `platform_info` | PlatformInfo | Platform information |
| `prediction_type` | string | Prediction type used |
| `result` | object | Prediction result (format varies by type) |

**`result` format for `expect_error`:**

```json
{
  "expected_runtime_ms": 138.5,
  "error_margin_ms": 12.3
}
```

**`result` format for `quantile`:**

```json
{
  "quantiles": {
    "0.5": 120.0,
    "0.9": 150.0,
    "0.95": 180.0,
    "0.99": 250.0
  }
}
```

**Error Responses**: `400` (invalid prediction type, type mismatch, invalid features, experiment mode error), `404` (model not found), `500` (prediction failure)

### WS /ws/predict

WebSocket endpoint for real-time streaming predictions. Accepts the same request format as `POST /predict` and returns the same response format. The connection stays open for multiple request-response cycles.

**Message format** (JSON over WebSocket):

- **Send**: `PredictionRequest` JSON
- **Receive**: `PredictionResponse` JSON or error object

Errors are sent as JSON messages without closing the connection, allowing the client to continue sending requests.

## Cache Management Endpoints

### GET /cache/stats

Get model cache performance statistics.

**Response** `200`

```json
{
  "cache_stats": {
    "hits": 42,
    "misses": 8,
    "size": 5,
    "max_size": 100,
    "hit_rate": 0.84
  },
  "message": "Cache statistics retrieved successfully"
}
```

### POST /cache/clear

Clear all cached models from memory. Does not affect models stored on disk.

**Response** `200`

```json
{
  "status": "success",
  "message": "Model cache cleared successfully"
}
```

## Shared Data Models

### PlatformInfo

Used across all training and prediction requests/responses.

| Field | Type | Required | Description |
|---|---|---|---|
| `software_name` | string | yes | Software platform name (e.g., `"vllm"`) |
| `software_version` | string | yes | Software version (e.g., `"0.4.0"`) |
| `hardware_name` | string | yes | Hardware platform name (e.g., `"NVIDIA A100"`) |

GPU specifications are automatically extracted from `hardware_name` when a known NVIDIA Tesla/datacenter GPU is detected. Supported GPUs: V100, V100-32GB, A10, A30, A40, A100, A100-80GB, T4, H100, H100-PCIe, H100-94GB, H20.

### Model Key Format

Models are stored with a key derived from the request parameters:

```
{model_id}__{software_name}-{software_version}__{hardware_name}
```

Example: `llama-7b__vllm-0.4.0__NVIDIA A100`

### Storage Format

Models are persisted as joblib files containing:

```json
{
  "predictor_state": {
    "model_config": {"input_dim": 2, "output_dim": 1, "hidden_layers": [64, 32]},
    "model_state_dict": "...",
    "feature_names": ["batch_size", "seq_len"],
    "feature_mean": [4.0, 288.0],
    "feature_std": [2.45, 153.0]
  },
  "metadata": {
    "model_id": "llama-7b",
    "platform_info": {"software_name": "vllm", "software_version": "0.4.0", "hardware_name": "NVIDIA A100"},
    "prediction_type": "expect_error",
    "samples_count": 10,
    "last_trained": "2026-01-30T12:00:00"
  }
}
```

## Error Handling

| Status Code | Meaning |
|---|---|
| `200` | Success |
| `400` | Invalid input (insufficient samples, invalid prediction type, feature errors) |
| `404` | Model not found |
| `422` | Request body validation error (Pydantic) |
| `500` | Internal error (training failure, prediction failure, storage error) |
| `503` | Service unhealthy (storage not accessible) |

Error response format:

```json
{
  "error": "Error category",
  "message": "Detailed error description",
  "details": {"additional": "context"}
}
```

HTTP exception format:

```json
{
  "detail": {
    "error": "Error category",
    "message": "Detailed error description"
  }
}
```
