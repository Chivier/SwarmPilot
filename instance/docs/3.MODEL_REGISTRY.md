# Model Registry Guide

## Overview

The Model Registry is a central configuration system that defines all available models and their deployment specifications. The instance uses this registry to locate and launch model containers.

## Directory Structure

The instance maintains a `dockers/` directory containing all model containers and the registry configuration:

```

├── easyocr_det/
│   ├── docker-compose.yaml
│   ├── Dockerfile
│   ├── start.sh

│   └── main.py
├── easyocr_rec/
│   ├── docker-compose.yaml
```

## Registry Configuration File

### `model_registry.yaml` Format

```yaml
models:
  - model_id: "easyocr/det"
    name: "EasyOCR Detection Model"
    directory: "easyocr_det"
    resource_requirements:
      cpu: "2"
      memory: "4Gi"
      gpu: false

  - model_id: "easyocr/rec"
    name: "EasyOCR Recognition Model"
    directory: "easyocr_rec"
    resource_requirements:
      cpu: "2"
      memory: "4Gi"
      gpu: false

  - model_id: "llm"
    name: "Large Language Model"
    directory: "llm"
    resource_requirements:
      cpu: "4"
      memory: "16Gi"
      gpu: true
      gpu_count: 1

  - model_id: "simple_mlp"
    name: "Simple Multi-Layer Perceptron"
    directory: "mlp"
    resource_requirements:
      cpu: "1"
      memory: "2Gi"
      gpu: false
```

## Field Definitions

### Model Entry Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `model_id` | string | Yes | Unique identifier used in API calls (e.g., "easyocr/det") |
| `name` | string | Yes | Human-readable model name |
| `directory` | string | Yes | Directory name under `dockers/` containing the model |
| `resource_requirements` | object | Yes | Resource allocation specifications |

### Resource Requirements Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `cpu` | string | Yes | Number of CPU cores (e.g., "2", "4") |
| `memory` | string | Yes | Memory allocation (e.g., "4Gi", "16Gi") |
| `gpu` | boolean | Yes | Whether GPU is required |
| `gpu_count` | integer | No | Number of GPUs (required if `gpu: true`) |

## Usage Workflow

### 1. Adding a New Model

1. Create model directory under `dockers/`:
   ```bash
   mkdir -p dockers/my_new_model
   ```

2. Implement required files (see [Model Container Specification](./2.MODEL_CONTAINER_SPEC.md))

3. Add entry to `model_registry.yaml`:
   ```yaml
   - model_id: "my/new_model"
     name: "My New Model"
     directory: "my_new_model"
     resource_requirements:
       cpu: "2"
       memory: "8Gi"
       gpu: false
   ```

4. Test the model:
   ```bash
   curl -X POST http://localhost:5000/model/start \
     -H "Content-Type: application/json" \
     -d '{"model_id": "my/new_model", "parameters": {}}'
   ```

### 2. Updating Model Configuration

To update resource requirements or other metadata:

1. Edit `model_registry.yaml`
2. Restart the instance (if configuration is cached)
3. Start the model with updated settings

### 3. Removing a Model

1. Remove the entry from `model_registry.yaml`
2. Optionally delete the model directory:
   ```bash
   rm -rf dockers/my_old_model
   ```

## Model ID Naming Convention

### Recommended Format

```
<namespace>/<model_name>
```

**Examples:**
- `easyocr/det` - EasyOCR detection component
- `easyocr/rec` - EasyOCR recognition component
- `openai/gpt4` - OpenAI GPT-4 model
- `custom/mlp` - Custom MLP model

### Simple Format (for single-namespace projects)

```
<model_name>
```

**Examples:**
- `llm`
- `simple_mlp`
- `detector`

## Resource Specification Guidelines

### CPU Allocation

- **Light models** (inference < 100ms): `"1"` core
- **Medium models** (inference 100ms-1s): `"2"` cores
- **Heavy models** (inference > 1s): `"4"` or more cores

### Memory Allocation

- **Small models** (< 1GB parameters): `"2Gi"`
- **Medium models** (1-10GB parameters): `"4Gi" - "8Gi"`
- **Large models** (> 10GB parameters): `"16Gi" - "32Gi"`

### GPU Configuration

For GPU-enabled models:
```yaml
resource_requirements:
  cpu: "4"
  memory: "16Gi"
  gpu: true
  gpu_count: 1  # or 2, 4, 8 for multi-GPU
```

For CPU-only models:
```yaml
resource_requirements:
  cpu: "2"
  memory: "4Gi"
  gpu: false
```

## Validation

### Registry Validation Script

Create a validation script to ensure registry integrity:

```python
import yaml
import os
from pathlib import Path

def validate_registry(registry_path: str, dockers_dir: str):
    """Validate model registry configuration."""
    with open(registry_path) as f:
        registry = yaml.safe_load(f)

    errors = []

    for model in registry["models"]:
        # Check required fields
        required = ["model_id", "name", "directory", "resource_requirements"]
        for field in required:
            if field not in model:
                errors.append(f"Model {model.get('model_id', 'unknown')} missing field: {field}")

        # Check directory exists
        model_dir = Path(dockers_dir) / model["directory"]
        if not model_dir.exists():
            errors.append(f"Directory not found: {model_dir}")

        # Check required files
        required_files = ["docker-compose.yaml", "Dockerfile", "start.sh", "pyproject.toml"]
        for file in required_files:
            if not (model_dir / file).exists():
                errors.append(f"Missing file in {model['directory']}: {file}")

    return errors

# Usage
errors = validate_registry("dockers/model_registry.yaml", "dockers")
if errors:
    print("Validation errors:")
    for error in errors:
        print(f"  - {error}")
else:
    print("Registry validation passed!")
```

## Best Practices

1. **Unique Model IDs**: Ensure each `model_id` is unique across the registry
2. **Consistent Naming**: Use clear, descriptive names for models and directories
3. **Resource Accuracy**: Set resource requirements based on actual model needs
4. **Version Control**: Keep `model_registry.yaml` under version control
5. **Documentation**: Add comments in the registry for complex configurations
6. **Testing**: Validate all models after registry updates

## Example: Complete Model Setup

```bash
# 1. Create model directory
mkdir -p dockers/my_detector

# 2. Create required files
cd dockers/my_detector
cat > Dockerfile <<EOF
FROM python:3.11-slim
RUN pip install uv
WORKDIR /app
COPY . .
RUN uv sync
CMD ["./start.sh"]
EOF

cat > start.sh <<EOF
#!/bin/bash
exec uv run python main.py
EOF
chmod +x start.sh

# Initialize uv project
uv init
uv add fastapi uvicorn

# 3. Update registry
cat >> ../model_registry.yaml <<EOF

  - model_id: "custom/detector"
    name: "Custom Object Detector"
    directory: "my_detector"
    resource_requirements:
      cpu: "2"
      memory: "4Gi"
      gpu: false
EOF
```
