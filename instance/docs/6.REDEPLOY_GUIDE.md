# Instance Redeployment Guide

This document describes how to perform instance redeployment for model updates, configuration changes, and maintenance operations.

> **See Also**: [API Reference](5.API_REFERENCE.md) for endpoint details

---

## Overview

Instance redeployment enables zero-downtime updates by:
1. Extracting pending tasks from the instance queue
2. Returning them to the scheduler for redistribution
3. Allowing the running task to complete
4. Updating the model/configuration
5. Re-registering with the scheduler

**Key Benefits:**
- No task loss during updates
- Running task completes normally
- Priority queue ordering preserved
- Automatic task redistribution

---

## Prerequisites

Before redeploying an instance:

1. **Multiple Instances**: Ensure at least one other instance exists for the same model_id
   - Scheduler needs a target for task redistribution
   - Minimum 2 instances recommended for same model

2. **Scheduler Coordination**: Instance must be registered with scheduler
   - Scheduler manages the redeployment workflow
   - Instance responds to redeployment requests

3. **Model Update Ready**: Have new model version/config prepared
   - Docker image pulled/built
   - Configuration files updated
   - Dependencies installed

## Redeployment Workflow

### Phase 1: Initiate Redeployment (Scheduler-Triggered)

The scheduler initiates redeployment by calling the instance's `/redeploy/start` endpoint:

**Endpoint:** `POST /redeploy/start`

**Request:**
```json
{
  "reason": "Updating to model v2.0",
  "target_model_id": "easyocr/dec-v2",
  "force": false
}
```

**Response:**
```json
{
  "success": true,
  "message": "Redeployment initiated. Returned 3 pending tasks.",
  "current_task": {
    "task_id": "task-123",
    "estimated_completion_ms": 2500
  },
  "pending_tasks": [
    {
      "task_id": "task-100",
      "model_id": "easyocr/dec",
      "task_input": {"image": "base64..."},
      "enqueue_time": 1699876543.123,
      "metadata": {}
    }
  ],
  "returned_count": 3
}
```

**What Happens:**
1. Instance sets `_is_redeploying = True` (stops accepting new tasks)
2. Extracts all pending tasks from queue (QUEUED status)
3. Preserves currently running task (if any)
4. Returns pending tasks to scheduler with full context

---

### Phase 2: Check Redeployment Status

Monitor redeployment progress using the status endpoint:

**Endpoint:** `GET /redeploy/status`

**Response:**
```json
{
  "success": true,
  "is_redeploying": true,
  "current_task": {
    "task_id": "task-123",
    "estimated_completion_ms": 1200
  },
  "message": "Redeploying"
}
```

**When to Check:**
- After initiating redeployment
- Before performing model update
- To verify current task completed

---

### Phase 3: Wait for Current Task

If a task was running when redeployment started:

**Do:**
- ✅ Wait for task to complete normally
- ✅ Monitor via `/redeploy/status`
- ✅ Check `estimated_completion_ms` for timing

**Don't:**
- ❌ Kill the running process
- ❌ Force container restart
- ❌ Manually cancel the task

**Example Monitoring:**
```bash
# Check status every second until no current_task
while true; do
  STATUS=$(curl -s http://localhost:8001/redeploy/status)
  CURRENT=$(echo $STATUS | jq -r '.current_task')

  if [ "$CURRENT" == "null" ]; then
    echo "No running task, safe to proceed"
    break
  fi

  echo "Task still running: $(echo $STATUS | jq -r '.current_task.task_id')"
  sleep 1
done
```

---

### Phase 4: Perform Model Update

Once the current task completes (or if there was no running task):

**Option A: Docker Container Update**
```bash
# Stop current container
docker stop swarmpilot-instance-gpu-1

# Pull new image
docker pull myregistry/easyocr:v2.0

# Start with new image
docker run -d --name swarmpilot-instance-gpu-1 \
  --gpus all \
  -p 8001:8001 \
  -v /data:/data \
  myregistry/easyocr:v2.0
```

**Option B: Model Hot-Swap (if supported)**
```bash
# Call instance's model management endpoint
curl -X POST http://localhost:8001/model/switch \
  -H "Content-Type: application/json" \
  -d '{
    "model_id": "easyocr/dec-v2",
    "model_path": "/models/easyocr-v2.0.pt",
    "force_reload": true
  }'
```

**Option C: Configuration Update**
```bash
# Update config file
vim /config/model.yaml

# Reload configuration
curl -X POST http://localhost:8001/config/reload
```

---

### Phase 5: Complete Redeployment (Instance-Side)

After model update, clear the redeploying flag:

**Endpoint:** `POST /redeploy/complete`

**Request:** (no body required)

**Response:**
```json
{
  "success": true,
  "message": "Redeployment completed successfully. Instance can now accept new tasks."
}
```

**What Happens:**
- Clears `_is_redeploying = False`
- Instance ready to accept new tasks locally

---

### Phase 6: Notify Scheduler

Finally, notify scheduler that redeployment is complete:

**Endpoint:** `POST http://scheduler:8000/instance/redeploy/complete`

**Request:**
```json
{
  "instance_id": "instance-gpu-1",
  "model_id": "easyocr/dec-v2",
  "endpoint": "http://192.168.1.100:8001",
  "platform_info": {
    "software_name": "docker",
    "software_version": "24.0",
    "hardware_name": "nvidia-rtx-3090"
  }
}
```

**Response:**
```json
{
  "success": true,
  "message": "Instance redeployment completed successfully. Instance is now ACTIVE.",
  "instance": {
    "instance_id": "instance-gpu-1",
    "model_id": "easyocr/dec-v2",
    "status": "active",
    ...
  }
}
```

**What Happens:**
- Scheduler updates instance status to ACTIVE
- Instance configuration updated in registry
- Instance can receive new task assignments

## Complete Automation Script

Here's a complete script to automate the redeployment process:

```bash
#!/bin/bash
set -e

INSTANCE_ID="instance-gpu-1"
INSTANCE_URL="http://localhost:8001"
SCHEDULER_URL="http://localhost:8000"
NEW_MODEL_ID="easyocr/dec-v2"
NEW_IMAGE="myregistry/easyocr:v2.0"

echo "=== Instance Redeployment Script ==="
echo "Instance: $INSTANCE_ID"
echo "New Model: $NEW_MODEL_ID"

# Step 1: Initiate redeployment (triggered by scheduler)
echo -e "\n[1/6] Initiating redeployment via scheduler..."
curl -X POST $SCHEDULER_URL/instance/redeploy/start \
  -H "Content-Type: application/json" \
  -d "{
    \"instance_id\": \"$INSTANCE_ID\",
    \"redeploy_reason\": \"Automated update to $NEW_MODEL_ID\",
    \"target_model_id\": \"$NEW_MODEL_ID\"
  }"

# Step 2: Wait for current task to complete
echo -e "\n[2/6] Waiting for current task to complete..."
while true; do
  STATUS=$(curl -s $INSTANCE_URL/redeploy/status)
  CURRENT=$(echo $STATUS | jq -r '.current_task')

  if [ "$CURRENT" == "null" ]; then
    echo "✓ No running task"
    break
  fi

  TASK_ID=$(echo $STATUS | jq -r '.current_task.task_id')
  EST_TIME=$(echo $STATUS | jq -r '.current_task.estimated_completion_ms')
  echo "  Task $TASK_ID still running (est. ${EST_TIME}ms remaining)"
  sleep 2
done

# Step 3: Perform model update
echo -e "\n[3/6] Updating model..."
docker stop $INSTANCE_ID || true
docker rm $INSTANCE_ID || true
docker pull $NEW_IMAGE

docker run -d --name $INSTANCE_ID \
  --gpus all \
  -p 8001:8001 \
  -v /data:/data \
  $NEW_IMAGE

# Wait for instance to be ready
echo "  Waiting for instance to start..."
for i in {1..30}; do
  if curl -s $INSTANCE_URL/health > /dev/null 2>&1; then
    echo "✓ Instance is healthy"
    break
  fi
  sleep 1
done

# Step 4: Complete redeployment on instance
echo -e "\n[4/6] Completing redeployment on instance..."
curl -X POST $INSTANCE_URL/redeploy/complete

# Step 5: Notify scheduler
echo -e "\n[5/6] Notifying scheduler..."
curl -X POST $SCHEDULER_URL/instance/redeploy/complete \
  -H "Content-Type: application/json" \
  -d "{
    \"instance_id\": \"$INSTANCE_ID\",
    \"model_id\": \"$NEW_MODEL_ID\",
    \"endpoint\": \"$INSTANCE_URL\",
    \"platform_info\": {
      \"software_name\": \"docker\",
      \"software_version\": \"24.0\",
      \"hardware_name\": \"nvidia-rtx-3090\"
    }
  }"

# Step 6: Verify
echo -e "\n[6/6] Verifying instance status..."
INSTANCE_INFO=$(curl -s "$SCHEDULER_URL/instance/info?instance_id=$INSTANCE_ID")
STATUS=$(echo $INSTANCE_INFO | jq -r '.instance.status')

if [ "$STATUS" == "active" ]; then
  echo "✓ Redeployment complete! Instance is ACTIVE"
else
  echo "✗ Warning: Instance status is $STATUS"
  exit 1
fi

echo -e "\n=== Redeployment Successful ==="
```

**Usage:**
```bash
chmod +x redeploy_instance.sh
./redeploy_instance.sh
```

## Task Queue Behavior

### Before Redeployment
```
Instance Queue:
┌─────────────────────────┐
│ RUNNING: task-123       │ ← Will complete
├─────────────────────────┤
│ QUEUED:  task-100       │ ↓
│ QUEUED:  task-101       │ ↓ Extracted
│ QUEUED:  task-102       │ ↓
└─────────────────────────┘
```

### During Redeployment
```
Instance State: REDEPLOYING
- New tasks rejected (returns error)
- Running task continues normally
- Queue is empty (tasks redistributed)
```

### After Redeployment
```
Instance Queue:
┌─────────────────────────┐
│ (Empty - ready for new) │
└─────────────────────────┘

Instance State: ACTIVE
- Accepting new tasks ✓
- Scheduler assigning tasks ✓
```

## Priority Queue Preservation

### enqueue_time Field

Each task has an `enqueue_time` (Unix timestamp) that determines its priority:

**During Normal Operation:**
```python
# Task submission automatically assigns enqueue_time
task = Task(
    task_id="task-123",
    model_id="easyocr/dec",
    task_input={"image": "base64..."},
    enqueue_time=time.time()  # Current timestamp
)
```

**During Redeployment:**
```python
# Extracted tasks preserve original enqueue_time
returned_tasks = [
    {
        "task_id": "task-100",
        "enqueue_time": 1699876543.123,  # Original timestamp preserved
        ...
    }
]

# Redistributed tasks submitted with preserved enqueue_time
await submit_task(
    task_id="task-100",
    enqueue_time=1699876543.123  # Not current time!
)
```

**Result:**
- Tasks maintain their place in the global queue
- FIFO ordering preserved across instances
- Fair processing guarantee

## Error Handling

### Common Errors

**1. Already Redeploying**
```json
{
  "detail": "Instance is already in redeploying state"
}
```
**Solution:** Wait for current redeployment to complete or cancel it first

**2. Task Extraction Failed**
```json
{
  "detail": "Failed to extract pending tasks: Database connection error"
}
```
**Solution:** Check instance logs, verify database connectivity

**3. No Target Instances**
```
Scheduler logs: "No available instances for reassignment (model_id: easyocr/dec)"
```
**Solution:** Register additional instances before redeployment

### Recovery Procedures

**If redeployment gets stuck:**

```bash
# 1. Check current status
curl http://localhost:8001/redeploy/status

# 2. If is_redeploying=true but no current_task:
#    Manually complete local redeployment
curl -X POST http://localhost:8001/redeploy/complete

# 3. Notify scheduler
curl -X POST http://localhost:8000/instance/redeploy/complete \
  -H "Content-Type: application/json" \
  -d '{ ... }'

# 4. Verify instance is ACTIVE
curl "http://localhost:8000/instance/info?instance_id=instance-gpu-1"
```

**If tasks were lost:**

```bash
# Check scheduler logs for failed redistributions
# Resubmit failed tasks manually if needed
```

## Best Practices

### Timing

✅ **Do:**
- Schedule redeployments during low-traffic periods
- Monitor queue size before initiating
- Stagger redeployments across instances
- Allow time for current task completion

❌ **Don't:**
- Redeploy during peak traffic
- Redeploy all instances simultaneously
- Force-kill running processes
- Skip status checks

### Capacity Planning

- **Minimum Instances**: Always have N+1 instances for critical models
- **Queue Management**: Keep queue sizes manageable (<100 tasks)
- **Load Distribution**: Ensure remaining instances can handle redistributed load

### Monitoring

```bash
# Pre-redeployment checks
curl http://localhost:8000/instance/list?model_id=easyocr/dec | jq '.count'
# Should return >= 2

curl "http://localhost:8000/instance/info?instance_id=instance-gpu-1" | jq '.stats.pending_tasks'
# Review queue size

# During redeployment
watch -n 1 'curl -s http://localhost:8001/redeploy/status | jq .'

# Post-redeployment validation
curl "http://localhost:8000/instance/info?instance_id=instance-gpu-1" | jq '.instance.status'
# Should be "active"
```

## Rollback Procedure

If the new model version has issues:

```bash
# 1. Initiate another redeployment
curl -X POST http://localhost:8000/instance/redeploy/start \
  -d '{"instance_id": "instance-gpu-1"}'

# 2. Rollback to previous version
docker stop instance-gpu-1
docker run -d --name instance-gpu-1 \
  myregistry/easyocr:v1.9  # Previous version

# 3. Complete redeployment with old model_id
curl -X POST http://localhost:8001/redeploy/complete

curl -X POST http://localhost:8000/instance/redeploy/complete \
  -d '{
    "instance_id": "instance-gpu-1",
    "model_id": "easyocr/dec",  # Original model_id
    ...
  }'
```
