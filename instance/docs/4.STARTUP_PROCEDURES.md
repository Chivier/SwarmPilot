## Step 1: Model Registry Lookup

### Process

1. Parse the `model_id` from the request body
2. Load and parse `dockers/model_registry.yaml`
3. Search for the matching model entry by `model_id`
4. Extract the model's `directory` and `resource_requirements`
5. Validate that the model directory exists in `dockers/`

### Implementation Example

```python
import yaml
from pathlib import Path

def lookup_model(model_id: str) -> dict:
    """Look up model configuration from registry."""
    # Load registry
    with open("dockers/model_registry.yaml") as f:
        registry = yaml.safe_load(f)

    # Find model
    for model in registry["models"]:
        if model["model_id"] == model_id:
            # Validate directory exists
            model_dir = Path("dockers") / model["directory"]
            if not model_dir.exists():
                raise ValueError(f"Model directory not found: {model_dir}")

            return model

    raise ValueError(f"Model not found in registry: {model_id}")

# Usage
model_config = lookup_model("easyocr/det")
# Returns: {
#   "model_id": "easyocr/det",
#   "directory": "easyocr_det",
#   "resource_requirements": {...}
# }
```

### Error Handling

| Error | HTTP Status | Response |
|-------|-------------|----------|
| Model ID not found in registry | 404 | `{"success": false, "error": "Model not found: {model_id}"}` |
| Model directory missing | 500 | `{"success": false, "error": "Model directory not found"}` |
| Registry file invalid | 500 | `{"success": false, "error": "Invalid registry configuration"}` |

## Step 2: Environment Variable Configuration

### Standard Environment Variables

The instance injects these standard variables into every model container:

```bash
MODEL_ID="easyocr/det"      # The requested model identifier
INSTANCE_ID="instance-001"  # The instance's unique ID
LOG_LEVEL="INFO"            # Logging level (INFO, DEBUG, ERROR)
```

### Custom Parameters

Parameters from the request's `parameters` field are converted to environment variables:

**Conversion Rules:**
1. Convert key to uppercase
2. Add `MODEL_` prefix
3. JSON-encode complex types (arrays, objects)
4. Simple types remain as strings

### Implementation Example

```python
import json
import os

def prepare_environment(model_id: str, instance_id: str, parameters: dict) -> dict:
    """Prepare environment variables for container."""
    env = {
        "MODEL_ID": model_id,
        "INSTANCE_ID": instance_id,
        "LOG_LEVEL": os.getenv("LOG_LEVEL", "INFO")
    }

    # Add custom parameters
    for key, value in parameters.items():
        env_key = f"MODEL_{key.upper()}"

        # JSON-encode complex types
        if isinstance(value, (list, dict)):
            env[env_key] = json.dumps(value)
        else:
            env[env_key] = str(value)

    return env

# Usage
env_vars = prepare_environment(
    model_id="simple_mlp",
    instance_id="instance-001",
    parameters={"layers": [32, 64], "device": "cuda"}
)
# Returns: {
#   "MODEL_ID": "simple_mlp",
#   "INSTANCE_ID": "instance-001",
#   "LOG_LEVEL": "INFO",
#   "MODEL_LAYERS": "[32, 64]",
#   "MODEL_DEVICE": "cuda"
# }
```

## Step 3: Docker Compose Configuration

### Port Mapping Formula

```
Model Container Port = Instance Port + 1000
```

**Example:**
- Instance runs on port `5000`
- Model container port: `5000 + 1000 = 6000`
- Mapping: `6000:8000` (host 6000 → container internal 8000)

### Configuration Generation

```python
import yaml

def generate_docker_compose(
    model_dir: str,
    instance_port: int,
    env_vars: dict,
    resources: dict
) -> str:
    """Generate docker-compose.yaml content."""
    model_port = instance_port + 1000

    compose_config = {
        "version": "3.8",
        "services": {
            "model": {
                "build": ".",
                "ports": [f"{model_port}:8000"],
                "environment": env_vars,
                "deploy": {
                    "resources": {
                        "limits": {
                            "cpus": resources["cpu"],
                            "memory": resources["memory"]
                        },
                        "reservations": {
                            "cpus": str(int(resources["cpu"]) // 2),
                            "memory": resources["memory"]
                        }
                    }
                }
            }
        }
    }

    # Add GPU configuration if needed
    if resources.get("gpu"):
        compose_config["services"]["model"]["deploy"]["resources"]["reservations"]["devices"] = [
            {
                "driver": "nvidia",
                "count": resources.get("gpu_count", 1),
                "capabilities": ["gpu"]
            }
        ]

    return yaml.dump(compose_config)
```

### Example Generated Configuration

```yaml
version: '3.8'
services:
  model:
    build: .
    ports:
      - "6000:8000"  # instance_port(5000) + 1000 : 8000
    environment:
      MODEL_ID: "easyocr/det"
      INSTANCE_ID: "instance-001"
      LOG_LEVEL: "INFO"
    deploy:
      resources:
        limits:
          cpus: '2'
          memory: 4Gi
        reservations:
          cpus: '1'
          memory: 4Gi
```

## Step 4: Container Launch

### Launch Process

1. Navigate to model directory
2. Write generated `docker-compose.yaml` (or modify existing)
3. Execute `docker-compose up -d --build`
4. Capture container ID for management
5. Monitor startup logs

### Implementation Example

```python
import subprocess
import os

def launch_container(model_dir: str, compose_content: str) -> str:
    """Launch model container using Docker Compose."""
    # Write docker-compose configuration
    compose_path = os.path.join(model_dir, "docker-compose.yaml")
    with open(compose_path, "w") as f:
        f.write(compose_content)

    # Launch container
    result = subprocess.run(
        ["docker-compose", "up", "-d", "--build"],
        cwd=model_dir,
        capture_output=True,
        text=True
    )

    if result.returncode != 0:
        raise RuntimeError(f"Container launch failed: {result.stderr}")

    # Get container ID
    result = subprocess.run(
        ["docker-compose", "ps", "-q"],
        cwd=model_dir,
        capture_output=True,
        text=True
    )

    container_id = result.stdout.strip()
    return container_id
```

## Step 5: Health Check and Verification

### Health Check Process

1. Construct health endpoint URL: `http://localhost:{instance_port+1000}/health`
2. Poll endpoint with exponential backoff
3. Maximum retry time: 30 seconds
4. Success: Model is ready
5. Failure: Stop container and return error

### Implementation Example

```python
import requests
import time

def wait_for_health(instance_port: int, max_wait: int = 30) -> bool:
    """Wait for model container to become healthy."""
    model_port = instance_port + 1000
    health_url = f"http://localhost:{model_port}/health"

    start_time = time.time()
    attempt = 0

    while time.time() - start_time < max_wait:
        try:
            response = requests.get(health_url, timeout=2)
            if response.status_code == 200:
                return True
        except requests.RequestException:
            pass

        # Exponential backoff: 0.5s, 1s, 2s, 4s, 8s, ...
        wait_time = min(0.5 * (2 ** attempt), 5)
        time.sleep(wait_time)
        attempt += 1

    return False

# Usage
if wait_for_health(instance_port=5000):
    print("Model is healthy and ready")
else:
    print("Model failed to start")
    # Clean up container
```

### Retry Strategy

| Attempt | Wait Time | Cumulative Time |
|---------|-----------|-----------------|
| 1 | 0.5s | 0.5s |
| 2 | 1.0s | 1.5s |
| 3 | 2.0s | 3.5s |
| 4 | 4.0s | 7.5s |
| 5 | 5.0s | 12.5s |
| 6 | 5.0s | 17.5s |
| 7+ | 5.0s | ... |

## Step 6: State Management

### State Update

Once the health check succeeds, update the instance's internal state:

```python
from datetime import datetime

def update_instance_state(
    model_id: str,
    container_id: str,
    parameters: dict,
    instance_port: int
) -> dict:
    """Update instance state after successful model start."""
    model_port = instance_port + 1000

    state = {
        "current_model": {
            "model_id": model_id,
            "started_at": datetime.utcnow().isoformat() + "Z",
            "parameters": parameters,
            "endpoint": f"http://localhost:{model_port}",
            "container_id": container_id
        }
    }

    return state
```

## Complete Startup Function

### Orchestration Example

```python
async def start_model(model_id: str, parameters: dict, instance_id: str, instance_port: int):
    """Complete model startup procedure."""
    try:
        # Step 1: Registry lookup
        model_config = lookup_model(model_id)

        # Step 2: Environment setup
        env_vars = prepare_environment(model_id, instance_id, parameters)

        # Step 3: Docker configuration
        model_dir = f"dockers/{model_config['directory']}"
        compose_content = generate_docker_compose(
            model_dir,
            instance_port,
            env_vars,
            model_config["resource_requirements"]
        )

        # Step 4: Container launch
        container_id = launch_container(model_dir, compose_content)

        # Step 5: Health check
        if not wait_for_health(instance_port):
            # Cleanup on failure
            subprocess.run(["docker-compose", "down"], cwd=model_dir)
            raise RuntimeError("Model failed health check")

        # Step 6: State management
        state = update_instance_state(model_id, container_id, parameters, instance_port)

        return {
            "success": True,
            "message": "Model started successfully",
            "model_id": model_id,
            "status": "running"
        }

    except Exception as e:
        return {
            "success": False,
            "error": str(e)
        }
```

## Error Handling Summary

| Stage | Error Type | Recovery Action |
|-------|------------|-----------------|
| Registry Lookup | Model not found | Return 404 error |
| Environment Setup | Invalid parameters | Return 400 error |
| Docker Config | Config generation failed | Return 500 error |
| Container Launch | Docker command failed | Return 500 error with logs |
| Health Check | Timeout | Stop container, return 500 error |
| State Management | State update failed | Log error, mark instance unhealthy |
