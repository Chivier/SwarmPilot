# Instance Service API Reference

## Base URL

```
http://<instance_host>:<instance_port>
```

Example: `http://localhost:5000`

## Authentication

Currently, no authentication is required. Future versions may implement token-based authentication.

---

## Model Management APIs

### POST `/model/start`

Start a model/tool on this instance. Each instance can only serve one model at a time.

**Request Body:**
```json
{
  "model_id": "string",           // Unique identifier of the model/tool to start
  "parameters": {                 // Model-specific initialization parameters (optional)
    "key": "value"
  }
}
```

**Example Requests:**
```json
{
  "model_id": "easyocr/det",
  "parameters": {}
}
```

```json
{
  "model_id": "simple_mlp",
  "parameters": {
    "layers": [32, 32, 32, 64, 64, 64, 32, 32]
  }
}
```

**Response (200 OK):**
```json
{
  "success": true,
  "message": "Model started successfully",
  "model_id": "string",
  "status": "running"
}
```

**Response (400 Bad Request):**
```json
{
  "success": false,
  "error": "Another model is already running. Stop the current model first."
}
```

**Response (500 Internal Server Error):**
```json
{
  "success": false,
  "error": "Failed to start model: [error details]"
}
```

---

### GET `/model/stop`

Stop the currently running model on this instance.

**Request Body:**
None

**Response (200 OK):**
```json
{
  "success": true,
  "message": "Model stopped successfully",
  "model_id": "string"
}
```

**Response (400 Bad Request):**
```json
{
  "success": false,
  "error": "No model is currently running"
}
```

## Task Management APIs

### POST `/task/submit`

Submit a new task to the task queue. Tasks are processed sequentially in FIFO order.

**Request Body:**
```json
{
  "task_id": "string",            // Unique identifier for this task
  "model_id": "string",           // Model/tool ID to use for this task
  "task_input": {                 // Model-specific input data
    "key": "value"
  }
}
```

**Example Request:**
```json
{
  "task_id": "task-001",
  "model_id": "easyocr/det",
  "task_input": {
    "image_url": "https://example.com/image.png",
    "language": "en"
  }
}
```

**Response (200 OK):**
```json
{
  "success": true,
  "message": "Task submitted successfully",
  "task_id": "string",
  "status": "queued",
  "position": 3                   // Position in the queue
}
```

**Response (400 Bad Request):**
```json
{
  "success": false,
  "error": "Task with this ID already exists"
}
```

**Response (422 Unprocessable Entity):**
```json
{
  "success": false,
  "error": "Model ID does not match the currently running model"
}
```

---

### GET `/task/list`

List all tasks and their current status in the queue.

**Request Body:**
None

**Query Parameters:**
- `status` (optional): Filter by status (`queued`, `running`, `completed`, `failed`)
- `limit` (optional): Maximum number of tasks to return (default: 100)

**Response (200 OK):**
```json
{
  "success": true,
  "total": 5,
  "tasks": [
    {
      "task_id": "string",
      "model_id": "string",
      "status": "completed",       // Status: queued, running, completed, failed
      "submitted_at": "2025-10-31T10:30:00Z",
      "started_at": "2025-10-31T10:30:05Z",
      "completed_at": "2025-10-31T10:30:15Z",
      "result": {                  // Present only if status is "completed"
        "output": "..."
      },
      "error": null                // Present only if status is "failed"
    }
  ]
}
```

---

### GET `/task/{task_id}`

Get detailed information about a specific task.

**Path Parameters:**
- `task_id`: The unique identifier of the task

**Response (200 OK):**
```json
{
  "success": true,
  "task": {
    "task_id": "string",
    "model_id": "string",
    "status": "running",
    "task_input": {
      "key": "value"
    },
    "submitted_at": "2025-10-31T10:30:00Z",
    "started_at": "2025-10-31T10:30:05Z",
    "completed_at": null,
    "result": null,
    "error": null
  }
}
```

**Response (404 Not Found):**
```json
{
  "success": false,
  "error": "Task not found"
}
```

---

### DELETE `/task/{task_id}`

Cancel a queued task or remove a completed/failed task from the list.

**Path Parameters:**
- `task_id`: The unique identifier of the task

**Response (200 OK):**
```json
{
  "success": true,
  "message": "Task cancelled/removed successfully",
  "task_id": "string"
}
```

**Response (400 Bad Request):**
```json
{
  "success": false,
  "error": "Cannot cancel a running task"
}
```

**Response (404 Not Found):**
```json
{
  "success": false,
  "error": "Task not found"
}
```

## Management APIs

### GET `/info`

Get current status and information about this instance.

**Request Body:**
None

**Response (200 OK):**
```json
{
  "success": true,
  "instance": {
    "instance_id": "string",      // Unique identifier for this instance
    "status": "running",           // Status: idle, running, busy, error
    "current_model": {             // null if no model is running
      "model_id": "string",
      "started_at": "2025-10-31T10:00:00Z",
      "parameters": {
        "key": "value"
      }
    },
    "task_queue": {
      "total": 5,
      "queued": 3,
      "running": 1,
      "completed": 1,
      "failed": 0
    },
    "uptime": 86400,               // Uptime in seconds
    "version": "1.0.0"
  }
}
```

---

### GET `/health`

Health check endpoint for monitoring and load balancing.

**Request Body:**
None

**Response (200 OK):**
```json
{
  "status": "healthy",
  "timestamp": "2025-10-31T10:30:00Z"
}
```

**Response (503 Service Unavailable):**
```json
{
  "status": "unhealthy",
  "error": "Model crashed or instance not responding",
  "timestamp": "2025-10-31T10:30:00Z"
}
```

## Usage Examples

### Starting a Model

```bash
curl -X POST http://localhost:5000/model/start \
  -H "Content-Type: application/json" \
  -d '{
    "model_id": "easyocr/det",
    "parameters": {}
  }'
```

### Submitting a Task

```bash
curl -X POST http://localhost:5000/task/submit \
  -H "Content-Type: application/json" \
  -d '{
    "task_id": "task-001",
    "model_id": "easyocr/det",
    "task_input": {
      "image_url": "https://example.com/image.png"
    }
  }'
```

### Checking Task Status

```bash
curl http://localhost:5000/task/task-001
```

### Listing All Tasks

```bash
curl "http://localhost:5000/task/list?status=completed&limit=10"
```

### Getting Instance Info

```bash
curl http://localhost:5000/info
```

### Health Check

```bash
curl http://localhost:5000/health
```

---

## Error Response Format

All error responses follow this standard format:

```json
{
  "success": false,
  "error": "Descriptive error message"
}
```

## HTTP Status Codes

| Code | Meaning | Usage |
|------|---------|-------|
| 200 | OK | Request succeeded |
| 400 | Bad Request | Invalid request parameters |
| 404 | Not Found | Resource not found |
| 422 | Unprocessable Entity | Valid request but cannot be processed |
| 500 | Internal Server Error | Server-side error |
| 503 | Service Unavailable | Service is temporarily unavailable |

## Rate Limiting

Currently, no rate limiting is implemented. Future versions may add rate limiting to prevent abuse.

## WebSocket Support (Future)

Future versions may support WebSocket connections for real-time task status updates:

```javascript
const ws = new WebSocket('ws://localhost:5000/ws/task/task-001');
ws.onmessage = (event) => {
  const status = JSON.parse(event.data);
  console.log('Task status:', status);
};
```
